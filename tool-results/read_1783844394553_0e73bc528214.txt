     8→## 1. physics-matter ⚡ (HIGH-PRIORITY AUDIT TARGET)
     9→
    10→### Critical rules
    11→1. **Matter body position = center of mass**, not top-left (unlike Arcade). `this.matter.add.sprite(x, y, 't')` places the body's center at `(x, y)`.
    12→2. **`setBody` / `setRectangle` / `setCircle` / `setPolygon` RESET ALL PROPERTIES** — mass, friction, collision filters, callbacks are all wiped. Re-apply after any shape change.
    13→3. **Force values are tiny** (`0.01`–`0.1`); velocity values are `1`–`15`. They are NOT pixel-based. `applyForce({x: 0.05, y: 0})` is correct; `applyForce({x: 50, y: 0})` will rocket the body.
    14→4. **`collisionFilter.group` overrides category/mask.** Same positive group = always collide; same negative group = never collide; `0`/different = use category/mask. Sensors still require matching filters to fire events.
    15→5. **Constraints must target the parent body**, never compound body `parts`. 32 collision categories max.
    16→
    17→### Right way (compound body + collision callback)
    18→```js
    19→const compound = this.matter.body.create({ parts: [partA, partB] });
    20→const player = this.matter.add.sprite(400, 200, 'hero');
    21→player.setExistingBody(compound);          // re-apply mass/friction/etc. after this
    22→player.setOnCollide((pair) => { /* pair.bodyA, pair.bodyB */ });
    23→
    24→// Collision categories
    25→const PLAYER = this.matter.world.nextCategory();  // 0x0002
    26→const ENEMY  = this.matter.world.nextCategory();
    27→player.setCollisionCategory(PLAYER);
    28→player.setCollidesWith([ENEMY]);
    29→```
    30→
    31→### Pitfalls
    32→- **DO NOT** call `setBody()` and assume collision filters persist — they're reset.
    33→- **DO NOT** target `compound.parts[i]` in `add.constraint()` — use the parent.
    34→- **DO NOT** forget `setSleepEvents(true, true)` if you want `sleepstart`/`sleepend` to fire.
    35→- **DO NOT** call `convertTilemapLayer(layer)` before `layer.setCollisionByProperty(...)` — collision must be set first.
    36→- **DO NOT** expect `restitution` to average — `Math.max(bodyA.restitution, bodyB.restitution)` wins.
    37→- `body.ignorePointer = true` prevents the mouseSpring from dragging that body.
    38→
    39→### v3→v4
    40→- No major Matter API change; `MatterGameObject` injection via `this.matter.add.gameObject(go, opts)` still works.

    44→## 2. input-keyboard-mouse-touch ⚡ (HIGH-PRIORITY AUDIT TARGET)
    45→
    46→### Critical rules
    47→1. **`setInteractive()` MUST be called before listening for Game Object pointer events.** Without it, `sprite.on('pointerdown', ...)` will never fire.
    48→2. **Pointer events fire at 3 levels** — Game Object (`pointerdown`), Scene per-object (`gameobjectdown`), Scene global (`pointerdown`). Higher handlers can `event.stopPropagation()`.
    49→3. **`pointer.worldX/worldY` are only valid inside input handlers.** Outside, call `pointer.updateWorldPoint(camera)` first.
    50→4. **`topOnly` defaults to `true`** — only the top-most interactive object receives events. Set `this.input.topOnly = false` to receive on all objects under the pointer.
    51→5. **Gamepad callbacks differ between plugin and instance.** Plugin-level: `(pad, button, value)`. Per-pad: `(buttonIndex, value, button)`. Mixing these is a classic bug.
    52→
    53→### Right way (interactive sprite + keyboard)
    54→```js
    55→sprite.setInteractive({ useHandCursor: true });
    56→sprite.on('pointerdown', (pointer, localX, localY, event) => {
    57→  // localX/localY are relative to the GO's top-left
    58→  event.stopPropagation();
    59→});
    60→
    61→// Keyboard with capture (prevents browser scroll on arrow keys)
    62→this.input.keyboard.addCapture('SPACE,UP,DOWN,LEFT,RIGHT');
    63→const space = this.input.keyboard.addKey('SPACE');
    64→// One-shot detection
    65→if (Phaser.Input.Keyboard.JustDown(space)) { /* fire once */ }
    66→```
    67→
    68→### Pitfalls
    69→- **DO NOT** use `pixelPerfect: true` for many objects — expensive (per-pixel alpha test). Prefer geometric hit areas.
    70→- **DO NOT** forget `this.input.setDraggable(sprite)` (or `{draggable: true}` in `setInteractive`) — `setInteractive()` alone does NOT enable drag.
    71→- **DO NOT** assume `pointer.x/y` are world coords — they are screen coords. Use `pointer.worldX/Y` or `cam.getWorldPoint()`.
    72→- **DO NOT** assume gamepad is connected on scene start — check `this.input.gamepad.total > 0` and use `pad1`–`pad4`. `'connected'` only fires for NEW connections.
    73→- **DO NOT** expect touch events during `preload()` — they are not dispatched until `create`.
    74→- Browser extensions (Vimium) can intercept keys — verify when debugging.
    75→- Gamepad API requires HTTPS in modern browsers.
    76→
    77→### v3→v4
    78→- No major changes; `Button` class accepts `isPressed` parameter for cross-scene-transition state correctness.

    82→## 3. text-and-bitmaptext ⚡ (HIGH-PRIORITY AUDIT TARGET)
    83→
    84→### Critical rules
    85→1. **Canvas `Text` re-renders the entire texture on every `setText`/`setStyle`/`setColor`/`setFontSize`.** Batch changes and call `updateText()` once. For frequently-updated text (scores, timers, damage numbers), use **BitmapText**.
    86→2. **Text origin defaults to (0,0)** — top-left. Sprites default to (0.5, 0.5). Call `text.setOrigin(0.5)` to center.
    87→3. **`setRTL(true)` MUST be called BEFORE `setText()`** — RTL changes how the canvas is constructed.
    88→4. **BitmapText font must be in cache** — load via `this.load.bitmapFont(key, png, xml/json)`. Missing font = console warning + undefined behavior.
    89→5. **`setCharacterTint`/`setWordTint`/`setDropShadow` are WebGL only** — no effect in Canvas renderer.
    90→
    91→### Right way (frequently updated text)
    92→```js
    93→// BAD for score updates each frame:
    94→scoreText.setText('Score: ' + score);  // re-creates canvas + re-uploads GPU texture
    95→
    96→// GOOD: use BitmapText
    97→this.load.bitmapFont('pixelFont', 'font.png', 'font.xml');
    98→const scoreText = this.add.bitmapText(10, 10, 'pixelFont', 'Score: 0', 24);
    99→scoreText.setText('Score: ' + score);  // only updates vertex data
   100→```
   101→
   102→### Pitfalls
   103→- **DO NOT** call `setColor()` thinking it will only change color — it triggers a full canvas rebuild.
   104→- **DO NOT** expect `align: 'center'` to center single-line text — it only affects multi-line. Use `setOrigin(0.5)` for single-line centering.
   105→- **DO NOT** use non-zero `letterSpacing` on Text for large bodies of text — each character is rendered individually. Use BitmapText instead.
   106→- **DO NOT** use `setDropShadow` on DynamicBitmapText or in Canvas renderer — no effect.
   107→- **DO NOT** forget to quote font names with digits/special chars: `fontFamily: '"Press Start 2P"'`.
   108→- `setFill()` and `setColor()` are equivalent aliases.
   109→
   110→### v3→v4
   111→- No breaking changes; BitmapText tinting now works correctly in v4.

   115→## 4. groups-and-containers ⚡ (HIGH-PRIORITY AUDIT TARGET)
   116→
   117→### Critical rules
   118→1. **Container is EXCLUSIVE by default** — adding a child to a new Container removes it from its previous one. Use `container.setExclusive(false)` to override.
   119→2. **`Container.setScrollFactor(x, y, updateChildren)` does NOT propagate to children by default.** Pass `true` as the third arg: `container.setScrollFactor(0, 0, true)`.
   120→3. **Group is NOT on the display list** — moving a Group does nothing visually. Use Container for transform inheritance.
   121→4. **Container origin is always (0,0)** — position children relative to (0,0); the origin cannot be changed.
   122→5. **Container needs `setSize(w, h)` before `setInteractive()` works** — containers have no implicit size. Pixel-perfect hit testing does NOT work on Containers.
   123→
   124→### Right way (HUD pinned to camera)
   125→```js
   126→const hud = this.add.container(10, 10);
   127→hud.setScrollFactor(0, 0, true);  // CRITICAL: third arg = true propagates to children
   128→hud.add([healthBar, scoreText, livesIcon]);
   129→```
   130→
   131→### Pitfalls
   132→- **DO NOT** put physics bodies on Container children unless the Container is at (0,0) — bodies will be offset.
   133→- **DO NOT** mix `depth` and display-list reordering — `depth` takes priority after the next sort pass.
   134→- **DO NOT** expect `group.add(child)` to put the child on the display list — only `group.create()` (with `true` second arg) or `this.add.existing()` does.
   135→- **DO NOT** mask Container children individually in Canvas — only the Container itself can be masked (WebGL: masks stack).
   136→- **DO NOT** put a Layer inside a Container — Layer cannot go in Container (reverse is OK).
   137→- `killAndHide(obj)` does NOT remove from group — it only sets `active=false, visible=false` for reuse.
   138→- Container has matrix-math per child per frame — deep nesting multiplies cost.
   139→
   140→### v3→v4
   141→- No major changes. Layer is the preferred render-bucketing primitive; Container for transform inheritance.

   145→## 5. scenes ⚡ (HIGH-PRIORITY AUDIT TARGET)
   146→
   147→### Critical rules
   148→1. **All `this.scene.*` operations are QUEUED, not immediate.** `this.scene.start('X')` happens on the next SceneManager update — do not rely on X's state in the same frame.
   149→2. **`start()` SHUTS DOWN the calling scene.** Use `launch()` or `run()` if you want both running in parallel.
   150→3. **`switch()` sleeps the current scene; `start()` shuts it down.** `switch()` preserves state; `start()` triggers full SHUTDOWN.
   151→4. **Paused scenes STILL RENDER.** Use `sleep()` to stop both update and render.
   152→5. **Reset state in `init()`, NOT the constructor.** Constructor runs once per instantiation; `init()` runs every start/restart.
   153→
   154→### Right way (state reset + shutdown cleanup)
   155→```js
   156→class GameScene extends Phaser.Scene {
   157→  constructor() { super('GameScene'); }
   158→  init() { this.score = 0; this.gameOver = false; }  // resets every restart
   159→  create() {
   160→    // CRITICAL: clean up on shutdown to avoid stale refs on restart
   161→    this.events.once('shutdown', () => { this.enemies = []; });
   162→  }
   163→}
   164→```
   165→
   166→### Pitfalls
   167→- **DO NOT** overwrite `this.sys` — breaks everything.
   168→- **DO NOT** assume `switch()` resumes a paused scene — it RESTARTS. Use `run()` for smart resume/wake/start.
   169→- **DO NOT** expect sleeping scenes to be silent — they can still receive events from other scenes (e.g., global registry events).
   170→- **DO NOT** rely on constructor state resets — they only run once.
   171→- **DO NOT** forget that `pause()` keeps rendering; if you want a frozen scene, use `sleep()`.
   172→- Scenes update in **reverse array order**, render in **forward array order** (last scene in array renders on top).
   173→- `create` event fires AFTER `create()` returns.
   174→
   175→### v3→v4
   176→- No major changes; InjectionMap customization via `map` config still supported.

   180→## 6. cameras ⚡ (HIGH-PRIORITY AUDIT TARGET)
   181→
   182→### Critical rules
   183→1. **`this.cameras` is the CameraManager, NOT an array.** Array is `this.cameras.cameras`; main camera is `this.cameras.main`.
   184→2. **Viewport vs Scroll are independent.** `setViewport/setPosition/setSize` = on-screen rectangle. `setScroll/scrollX/scrollY` = what world area the camera looks at.
   185→3. **`cam.startFollow(target)` SNAPS on first call.** To avoid a visible snap, set scroll to the target's position first.
   186→4. **Effects do NOT restart by default.** Calling `cam.fade()` while a fade is running does nothing unless you pass `force: true`.
   187→5. **`pan()` OVERRIDES follow** while running — follow resumes after pan completes.
   188→
   189→### Right way (HUD camera + main camera with bounds)
   190→```js
   191→const main = this.cameras.main;
   192→main.setBounds(0, 0, 2048, 2048);
   193→main.startFollow(player, false, 0.1, 0.1);  // lerp 0.1 = smooth
   194→
   195→// HUD camera (does not scroll)
   196→const hudCam = this.cameras.add(0, 0, 800, 600).setName('hud');
   197→main.ignore(hudGroup);          // main cam doesn't draw HUD
   198→hudCam.ignore(worldGroup);      // HUD cam doesn't draw world
   199→
   200→// Convert pointer to world coords (essential with scroll/zoom)
   201→const wp = main.getWorldPoint(pointer.x, pointer.y);
   202→```
   203→
   204→### Pitfalls
   205→- **DO NOT** set zoom to 0 — clamped to 0.001 internally but breaks rendering.
   206→- **DO NOT** use `setMask()` in WebGL — logs warning. Use `cam.filters.external.addMask()` instead. `setMask` is Canvas-only (GeometryMask).
   207→- **DO NOT** use non-integer zoom with `roundPixels: true` — causes jitter.
   208→- **DO NOT** forget to call `camControl.update(delta)` in `update()` for `FixedKeyControl`/`SmoothedKeyControl` — they do not auto-update.
   209→- **DO NOT** rotate the viewport — it's always axis-aligned; rotation is render-only.
   210→- **DO NOT** create more than 32 cameras if you need `ignore()` — cameras 33+ get ID 0 and cannot exclude objects.
   211→- `setBounds` only restricts scrolling — objects can still be placed outside.
   212→
   213→### v3→v4
   214→- **Camera matrix rewritten.** `Camera#matrix` now = rotation + zoom + scroll (no position). `matrixExternal` = position only. `matrixCombined` = `matrix * matrixExternal`. If you accessed matrices directly, update code.
   215→- Use `TransformMatrix.copyWithScrollFactorFrom(matrix, scrollX, scrollY, scrollFactorX, scrollFactorY)` instead of manually subtracting `camera.scrollX * src.scrollFactorX`.
   216→- Cameras have `filters.internal`/`filters.external` by default — no `enableFilters()` needed.
   217→

   220→## 7. game-object-components ⚡ (HIGH-PRIORITY AUDIT TARGET)
   221→
   222→### Critical rules
   223→1. **`alpha = 0`, `scale = 0`, `visible = false` ALL clear a render flag** — the object is NOT drawn at all (not just transparent).
   224→2. **`setTintFill()` is REMOVED in v4.** Use `sprite.setTint(color).setTintMode(Phaser.TintModes.FILL)`. Calling `setTintFill()` logs a console error.
   225→3. **`z` is NOT depth** — `transform.z` is a generic coordinate. Use `depth` (Depth component) for render order.
   226→4. **`Mask` component is Canvas-only in v4.** For WebGL masking, use `enableFilters()` then `filters.internal.addMask()`. `setMask()` logs a warning in WebGL.
   227→5. **Flip does NOT affect physics bodies** — it's rendering-only. Track facing direction separately in collision code.
   228→
   229→### Right way (v4 tint fill + WebGL mask)
   230→```js
   231→// v3: sprite.setTintFill(0xff0000);  // REMOVED in v4
   232→sprite.setTint(0xff0000).setTintMode(Phaser.TintModes.FILL);  // v4 correct
   233→
   234→// v4 WebGL mask (BitmapMask is removed)
   235→sprite.enableFilters();
   236→sprite.filters.internal.addMask(maskShape);
   237→```
   238→
   239→### Pitfalls
   240→- **DO NOT** use `setSize()` thinking it changes visual size — it changes internal/native dimensions. Use `setDisplaySize()` for visual size (adjusts scale).
   241→- **DO NOT** mix `depth` and `setToTop()`/`setAbove()` — `depth` wins after the next sort.
   242→- **DO NOT** apply physics bodies to objects with `scrollFactor != 1` — physics uses world position, scroll factor offsets rendering only.
   243→- **DO NOT** call `enableFilters()` lazily inside `update()` — call once in `create()`.
   244→- **DO NOT** expect Shape game objects to support `setTint()` — they don't. Use `setFillStyle(color, alpha)` / `setStrokeStyle(lineWidth, color, alpha)`.
   245→- Tint modes in v4: `MULTIPLY` (default), `FILL`, `ADD`, `SCREEN`, `OVERLAY`, `HARD_LIGHT`.
   246→
   247→### v3→v4
   248→- `setTintFill()` removed → `setTint(color).setTintMode(FILL)`.
   249→- Tint color and tint mode are now separate (`tintMode` property + `setTintMode()` method).
   250→- WebGL masking via filters; `setMask()` is Canvas-only.
   251→- New components: `Lighting`, `RenderSteps`, `RenderNodes`, `Filters`, `FilterList`.
   252→

   255→## 8. v3-to-v4-migration ⚡ (HIGH-PRIORITY AUDIT TARGET)
   256→
   257→### Critical rules (breaking changes)
   258→1. **FX (`preFX`/`postFX`) → Filters.** `sprite.preFX.addGlow(...)` → `sprite.enableFilters(); sprite.filters.internal.addGlow(...)`.
   259→2. **`BitmapMask` removed.** Use `filters.internal.addMask(maskObject)`. `GeometryMask` still exists (Canvas only).
   260→3. **`setTintFill()` removed.** Use `setTint(color).setTintMode(Phaser.TintModes.FILL)`.
   261→4. **`Geom.Point` removed.** Use `Phaser.Math.Vector2`. All geometry classes now return `Vector2` (Circle, Ellipse, Line, Polygon, Rectangle, Triangle).
   262→5. **`Math.TAU` value changed.** v3: `PI/2` (incorrect). v4: `PI*2` (correct). `Math.PI2` removed → use `Math.TAU`. New: `Math.PI_OVER_2` for `PI/2`.
   263→6. **`Phaser.Struct.Set` and `Phaser.Struct.Map` removed** → use native `Set` and `Map`. Methods like `iterateLocal`, `contains`, `setAll` are gone.
   264→7. **`DynamicTexture`/`RenderTexture` now require explicit `.render()`** — drawing commands are buffered, not executed immediately.
   265→8. **Pipelines → RenderNodes.** Custom WebGL pipelines must be rewritten as render nodes. Do NOT make direct `gl` calls — use an `Extern` game object if you need raw WebGL.
   266→9. **`Mesh` and `Plane` game objects REMOVED.**
   267→10. **`roundPixels` now defaults to `false`** (was `true` in v3). Per-object control via `gameObject.vertexRoundMode`.
   268→
   269→### Right way (FX → Filters, BitmapMask → Mask filter)
   270→```js
   271→// v3 FX:
   272→sprite.preFX.addGlow(0xff00ff, 4);
   273→sprite.postFX.addBlur(0, 2, 2, 1);
   274→
   275→// v4 Filters:
   276→sprite.enableFilters();
   277→sprite.filters.internal.addGlow(0xff00ff, 4, 0, 1);
   278→sprite.filters.external.addBlur(0, 2, 2, 1);
   279→
   280→// v3 BitmapMask:
   281→const mask = new Phaser.Display.Masks.BitmapMask(scene, maskImage);
   282→sprite.setMask(mask);
   283→// v4:
   284→sprite.enableFilters();
   285→sprite.filters.internal.addMask(maskImage);
   286→
   287→// v3 ColorMatrix:
   288→colorMatrix.sepia();
   289→// v4:
   290→colorMatrix.colorMatrix.sepia();
   291→
   292→// v3 lighting:
   293→sprite.setPipeline('Light2D');
   294→// v4:
   295→sprite.setLighting(true);
   296→```
   297→
   298→### Pitfalls / removed APIs
   299→- `Create.GenerateTexture` and `TextureManager.generate` removed.
   300→- `Math.SinCosTableGenerator` removed.
   301→- All legacy polyfills removed (Array.forEach, Array.isArray, console, Math.trunc, performance.now, requestAnimationFrame, Uint32Array).
   302→- Spine 3/4 bundled plugins no longer updated — use Esoteric Software's official plugin.
   303→- `DOMElement` now throws if it has no container.
   304→- `Shader#setTextures()` now REPLACES (not appends) — call once with the full array.
   305→- Compressed textures must be re-compressed with Y=0 at bottom (GL orientation).
   306→- Custom shaders: texture coordinates now use GL conventions (Y=0 at bottom).
   307→
   308→### Migration checklist (key items)
   309→- [ ] Replace `preFX`/`postFX` with `filters.internal`/`filters.external`
   310→- [ ] Replace `BitmapMask` with `FilterMask`
   311→- [ ] Replace `setTintFill()` with `setTint().setTintMode(FILL)`
   312→- [ ] Replace `Geom.Point` with `Vector2`
   313→- [ ] Replace `Math.PI2` with `Math.TAU`; `Math.TAU` (if expecting PI/2) with `Math.PI_OVER_2`
   314→- [ ] Replace `Phaser.Struct.Set/Map` with native `Set/Map`
   315→- [ ] Add `.render()` calls to `DynamicTexture`/`RenderTexture`
   316→- [ ] Re-compress compressed textures for new Y-axis orientation
   317→- [ ] Replace `sprite.setPipeline('Light2D')` with `sprite.setLighting(true)`
   318→- [ ] Remove `Mesh`/`Plane` usage
   319→- [ ] Replace derived FX (`Bloom`, `Shine`, `Circle`, `Gradient`) with Actions (`AddEffectBloom`, `AddEffectShine`, `AddMaskShape`) or GameObjects
   320→- [ ] Remove `TileSprite` texture cropping (no longer supported)

   324→## 9. v4-new-features ⚡ (HIGH-PRIORITY AUDIT TARGET)
   325→
   326→### Critical rules
   327→1. **Filters are WebGL only.** `enableFilters()` returns early on Canvas. Game objects REQUIRE `enableFilters()` first; cameras have filters by default.
   328→2. **Internal filters run BEFORE camera transform (object-local); External AFTER (screen space).** Internal is cheaper (object region only); External is full-screen.
   329→3. **Filter order matters** — each filter receives the output of the previous one.
   330→4. **`CaptureFrame` requires `camera.setForceComposite(true)`** — otherwise nothing is captured.
   331→5. **`SpriteGPULayer` requires single texture + single image per layer.** Populate once; updating buffer is expensive. "Remove" members by setting `scaleX/scaleY/alpha = 0`.
   332→
   333→### Right way (filters on sprite + camera)
   334→```js
   335→const sprite = this.add.sprite(400, 300, 'hero');
   336→sprite.enableFilters();  // MUST call before accessing .filters
   337→sprite.filters.internal.addGlow(0x00ff00, 4);  // object-local space
   338→
   339→// Cameras have filters by default — no enableFilters() needed
   340→this.cameras.main.filters.external.addBlur(1, 2, 2, 1);  // screen space
   341→```
   342→
   343→### Pitfalls
   344→- **DO NOT** try to change Glow `quality` or `distance` after creation — they're immutable. Destroy and recreate.
   345→- **DO NOT** expect a dedicated `Bloom` filter — use `ParallelFilters` (Threshold + Blur + ADD blend) or `Phaser.Actions.AddEffectBloom()`.
   346→- **DO NOT** update `SpriteGPULayer` members every frame — buffer rebuilds are expensive. Populate once, mutate `alpha`/`scale` to hide.
   347→- **DO NOT** use `TilemapGPULayer` for isometric/hex/staggered maps — orthographic only.
   348→- **DO NOT** forget `generateLayerDataTexture()` after editing tiles on `TilemapGPULayer` — changes won't appear otherwise.
   349→- Lighting changes the shader — breaks render batches. Group lit objects together.
   350→
   351→### New v4-only APIs
   352→- New GameObjects: `CaptureFrame`, `Gradient`, `Noise`, `NoiseCell2D/3D/4D`, `NoiseSimplex2D/3D`, `SpriteGPULayer`, `Stamp`, `TilemapGPULayer`.
   353→- New components: `Lighting`, `RenderSteps`, `RenderNodes`.
   354→- New tint modes: `MULTIPLY`, `FILL`, `ADD`, `SCREEN`, `OVERLAY`, `HARD_LIGHT`.
   355→- New filters: `Blend`, `Blocky`, `CombineColorMatrix`, `GradientMap`, `ImageLight`, `Key`, `Mask`, `NormalTools`, `PanoramaBlur`, `ParallelFilters`, `Quantize`, `Sampler`, `Threshold`.
   356→- `setRenderNodeRole(role, nodeName, data?)` to override render nodes per role (`'Submitter'`, `'Transformer'`, `'Texturer'`).

   360→## 10. game-setup-and-config ⚡ (HIGH-PRIORITY AUDIT TARGET)
   361→
   362→### Critical rules
   363→1. **Default dimensions changed in v4: width=1024, height=768** (was 800x600 in v3).
   364→2. **`scale` sub-object takes priority over top-level** `width`/`height`/`zoom`/`parent`. Don't set both — leads to confusion.
   365→3. **`Phaser.WEBGL` has NO fallback.** Unlike `AUTO`, if WebGL is unsupported the game fails silently. Use `Phaser.AUTO` for safety.
   366→4. **`parent: null` vs `parent: undefined`.** `undefined` (or omitted) appends to `document.body`. `null` = no parent — you must add the canvas to the DOM yourself.
   367→5. **`fps.target` is advisory only.** Use `fps.limit` to actually cap the frame rate. `fps.limit` can only SLOW DOWN, never speed up beyond the display refresh.
   368→
   369→### Right way (Matter + scale + pixel art)
   370→```js
   371→const config = {
   372→  type: Phaser.AUTO,             // WebGL with Canvas fallback
   373→  scale: {
   374→    parent: 'game-container',
   375→    mode: Phaser.Scale.FIT,
   376→    autoCenter: Phaser.Scale.CENTER_BOTH,
   377→    width: 1280,
   378→    height: 720,
   379→  },
   380→  physics: {
   381→    default: 'matter',
   382→    matter: {
   383→      gravity: { y: 1 },
   384→      enableSleeping: true,
   385→      debug: false,
   386→      setBounds: true,           // walls around canvas edges
   387→    },
   388→  },
   389→  input: { gamepad: true },      // enable gamepad
   390→  scene: [BootScene, GameScene],
   391→};
   392→```
   393→
   394→### Pitfalls
   395→- **DO NOT** set `transparent: true` and expect `backgroundColor` to work — `transparent` forces `rgba(0,0,0,0)`.
   396→- **DO NOT** use `Phaser.HEADLESS` for production — DOM still required; for unit tests only.
   397→- **DO NOT** use `smoothPixelArt` with Canvas renderer — WebGL only.
   398→- **DO NOT** forget `dom.createContainer: true` requires a `parent` element.
   399→- `Game.destroy()` is ASYNCHRONOUS — flags for destruction on the next frame. Listen for `DESTROY` event.
   400→- `window.FORCE_WEBGL` / `window.FORCE_CANVAS` globals override `config.type` (dev/test only).
   401→- `MatterWorldConfig` does NOT have `timing` field — it's `timing.timeScale` under `matter.timing.timeScale`? Actually, the worklog notes "Removed PhaserGame matter.timing (not in MatterWorldConfig)". Use `matter.runner.fps` for fixed timestep. Fields that exist: `gravity`, `setBounds`, `enableSleeping`, `positionIterations`, `velocityIterations`, `constraintIterations`, `timing.timeScale`, `autoUpdate`, `debug`, `runner`.
   402→- GameConfig does NOT have `contextCreation` (removed in v4) — pass your own context via `canvas` + `context` instead.

   414→## 11. tweens ⚡ (HIGH-PRIORITY AUDIT TARGET)
   415→
   416→### Critical rules
   417→1. **Tweens auto-destroy after completion** unless `persist: true`. Storing a reference and using it after completion = stale.
   418→2. **`repeat` is per-property; `loop` restarts the entire tween.** `repeat: 1` = property plays twice total. `loop: -1` means `onComplete` NEVER fires.
   419→3. **`onComplete` never fires for infinite tweens** — use `tween.completeAfterLoop(0)` to end gracefully.
   420→4. **Seeking suppresses events by default.** `tween.seek(ms)` does not dispatch events unless you pass `true` as the third arg.
   421→5. **Tweens skip properties starting with `_`** (underscore prefix). Use this to protect internal fields.
   422→
   423→### Right way (chain + relative values + callbacks)
   424→```js
   425→this.tweens.chain({
   426→  targets: this.player,
   427→  tweens: [
   428→    { x: 300, duration: 1000, ease: 'Power2' },
   429→    { y: '+=100', duration: 800, ease: 'Bounce.easeOut' },  // relative
   430→    { alpha: 0, duration: 400, onComplete: () => this.player.destroy() },
   431→  ],
   432→});
   433→
   434→// Stagger across multiple targets
   435→this.tweens.add({
   436→  targets: [s1, s2, s3],
   437→  alpha: 0,
   438→  duration: 500,
   439→  delay: this.tweens.stagger(100, { from: 'center' }),
   440→});
   441→```
   442→
   443→### Pitfalls
   444→- **DO NOT** use `this.tweens.stagger()` with a single target — no visible effect.
   445→- **DO NOT** forget to destroy a `persist: true` tween when done — it stays in memory.
   446→- **DO NOT** set `duration: 0` — internally clamped to 0.01ms minimum.
   447→- **DO NOT** rely on `onComplete` for `loop: -1` tweens — never fires.
   448→- **DO NOT** destroy a tween's target without stopping the tween first — tween auto-completes early if `target.isDestroyed === true`.
   449→- `TweenManager.timeScale` multiplies with `Tween.timeScale` — setting either to 0 freezes.
   450→- Callbacks receive `(tween, targets)` — not `(target)`.
   451→- Easing string names are case-insensitive; `'Power2'` = `'Cubic.easeOut'`. Bare names default to `.Out` variant.
   452→
   453→### v3→v4
   454→- No major changes to the Tween API.

   458→## 12. time-and-timers ⚡ (HIGH-PRIORITY AUDIT TARGET)
   459→
   460→### Critical rules
   461→1. **`repeat: N` means N+1 total fires** (1 initial + N repeats). Off-by-one source.
   462→2. **`delay: 0` with `repeat`/`loop` throws** `'TimerEvent infinite loop created via zero delay'`. Always use a non-zero delay.
   463→3. **`this.time.delayedCall(delay, callback)` is the safe one-shot pattern.** For repeating, use `addEvent({delay, callback, repeat})`.
   464→4. **Timelines start PAUSED.** You MUST call `timeline.play()` after `this.add.timeline([...])`.
   465→5. **`timer.reset(config)` does NOT re-add to the Clock.** You must call `this.time.addEvent(timer)` again.
   466→
   467→### Right way (delayed call + repeating timer + remove)
   468→```js
   469→// One-shot
   470→this.time.delayedCall(1000, () => this.scene.start('GameOver'));
   471→
   472→// Repeating (5 total fires: 1 initial + 4 repeats)
   473→const timer = this.time.addEvent({
   474→  delay: 500,
   475→  callback: this.fireBullet,
   476→  callbackScope: this,
   477→  repeat: 4,
   478→});
   479→
   480→// Stop
   481→timer.remove();          // silently
   482→timer.remove(true);      // fires callback one last time
   483→
   484→// Timeline (always starts paused)
   485→const tl = this.add.timeline([
   486→  { at: 0, run: () => this.title.setAlpha(1) },
   487→  { at: 1000, tween: { targets: this.title, y: 100, duration: 500 } },
   488→]);
   489→tl.play();
   490→```
   491→
   492→### Pitfalls
   493→- **DO NOT** omit `callbackScope` with a regular function — `this` defaults to the TimerEvent, not the scene. Use arrow functions or `callbackScope: this`.
   494→- **DO NOT** expect `timeline.timeScale` to affect its spawned tweens — only the timeline's own elapsed counter. Set tween `timeScale` separately.
   495→- **DO NOT** use `once: true` on a Timeline event and expect it to reappear on `reset()` or loop — it's spliced permanently.
   496→- **DO NOT** assume `addEvent()` is immediate — pushed to a pending list, active next frame.
   497→- **DO NOT** confuse `paused = true` with `timeScale = 0` — both freeze timers but `paused` skips the update loop entirely (prefer for full freeze).
   498→- Scene pause also pauses the Timeline.
   499→
   500→### v3→v4
   501→- No major changes.

   505→## 13. events-system ⚡ (HIGH-PRIORITY AUDIT TARGET)
   506→
   507→### Critical rules
   508→1. **`on()` does NOT auto-remove.** Every `scene.restart()` adds duplicate listeners — memory leak. Always clean up in `shutdown`:
   509→   ```js
   510→   this.events.once('shutdown', () => this.input.off('pointerdown', this.shoot, this));
   511→   ```
   512→2. **`off()` requires the EXACT same function reference AND context.** Anonymous arrow functions cannot be removed — store a reference or use class methods.
   513→3. **`this.game.events` and `this.events` are DIFFERENT emitters.** Game events persist across scene restarts; scene events don't. Clean up game.events listeners on SHUTDOWN.
   514→4. **`changedata-{key}` callback args differ from generic `changedata`.** Key-specific: `(parent, value, previousValue)`. Generic: `(parent, key, value, previousValue)`. Easy to miss.
   515→5. **Sleeping scenes can still receive events** from other scenes (global registry). Be careful with active listeners on sleeping scenes.
   516→
   517→### Right way (cleanup on shutdown + cross-scene communication)
   518→```js
   519→create() {
   520→  this.events.on('update', this.onUpdate, this);     // named method, this context
   521→  this.game.events.on('blur', this.onBlur, this);    // game-level persists!
   522→
   523→  this.events.once('shutdown', () => {
   524→    this.events.off('update', this.onUpdate, this);
   525→    this.game.events.off('blur', this.onBlur, this);  // CRITICAL
   526→  });
   527→}
   528→
   529→// Cross-scene via global registry (NOT scene events)
   530→this.registry.set('score', 100);                          // Scene A
   531→// Scene B:
   532→this.registry.events.on('changedata-score', (game, val) => this.scoreText.setText(val));
   533→// Cleanup registry listeners too:
   534→this.events.once('shutdown', () => this.registry.events.off('changedata-score', handler));
   535→```
   536→
   537→### Pitfalls
   538→- **DO NOT** use arrow literals for listeners you intend to remove: `this.input.on('pointerdown', () => this.shoot())` — cannot be removed.
   539→- **DO NOT** use `removeAllListeners()` without an arg unless you want to nuke all events on that emitter.
   540→- **DO NOT** confuse `shutdown` (scene stops, can restart) with `destroy` (permanent). Restart fires `SHUTDOWN` → `START` → `CREATE`, never `DESTROY`.
   541→- **DO NOT** forget that input events fire in order: Game Object → `gameobjectdown` on `this.input` → `pointerdown` on `this.input`. `stopPropagation()` halts further levels.
   542→- Named constants preferred: `Phaser.Scenes.Events.SHUTDOWN`, `Phaser.Input.Events.POINTER_DOWN`, etc.
   543→- Key-suffix pattern: `'filecomplete-image-logo'`, `'animationcomplete-walk'`, `'changedata-score'`.
   544→
   545→### v3→v4
   546→- No major changes; eventemitter3 still underlying.

   550→## 14. animations
   551→
   552→### Critical rules
   553→1. **Animations are GLOBAL by default.** `this.anims.create()` registers across all scenes. Do NOT recreate in every scene — it logs a warning and returns the existing one. Guard with `if (!this.anims.exists(key))`.
   554→2. **`repeat: -1` NEVER fires `animationcomplete`.** Use `stop()` to end infinite animations — fires `animationstop` instead.
   555→3. **`frameRate` BEATS `duration`.** If both are set, `frameRate` wins. Set only `duration` (leave `frameRate` null) to control total length.
   556→4. **Local animations override global.** Same key on a sprite's local map takes priority.
   557→5. **Mix delays only work with `play()`.** `playAfterDelay`/`playAfterRepeat` bypass mixes.
   558→
   559→### Right way (global anims + chained playback)
   560→```js
   561→// In BootScene or PreloadScene (once):
   562→if (!this.anims.exists('walk')) {
   563→  this.anims.create({
   564→    key: 'walk',
   565→    frames: this.anims.generateFrameNames('player', { prefix: 'walk_', start: 1, end: 8, zeroPad: 2 }),
   566→    frameRate: 12,
   567→    repeat: -1,
   568→  });
   569→}
   570→
   571→// In GameScene:
   572→sprite.play('walk');
   573→sprite.chain('idle');  // queued after walk stops
   574→```
   575→
   576→### Pitfalls
   577→- **DO NOT** expect `animationcomplete` for `repeat: -1` — use `animationstop`.
   578→- **DO NOT** set both `frameRate` and `duration` expecting duration to win — frameRate wins.
   579→- **DO NOT** forget per-frame `duration` is ADDITIVE on top of base msPerFrame.
   580→- **DO NOT** call `sprite.play(key)` thinking it queues — it STOPS the current anim (fires `animationstop`). Use `play(key, true)` to skip if already playing.
   581→- **DO NOT** leave chained anims unintentionally — clear with `sprite.anims.chain()` (no args).
   582→- Chained anims fire after `animationstop` too — clear queue before stopping if unwanted.
   583→- `generateFrameNumbers` end=-1 means last frame; `__BASE` is excluded automatically.
   584→
   585→### v3→v4
   586→- No breaking changes.

   590→## 15. audio-and-sound
   591→
   592→### Critical rules
   593→1. **Sounds are NOT auto-cleaned on scene shutdown.** Looping sounds continue across scene changes unless explicitly stopped. Stop them in `shutdown`.
   594→2. **Browser autoplay policy: audio is locked until user interaction.** Phaser handles this automatically; listen for `UNLOCKED` event if you need to know when ready.
   595→3. **Always provide multiple audio formats** — `this.load.audio('bgm', ['bgm.ogg', 'bgm.mp3'])`. MP3 broadest; OGG lacks Safari; AAC/M4A good for Safari/iOS.
   596→4. **HTML5 Audio needs `instances` for simultaneous playback** — `this.load.audio('shot', 'shot.mp3', { instances: 4 })`. Default is 1. WebAudio has no such limit.
   597→5. **Spatial audio is WebAudio only** — silently ignored on HTML5 Audio fallback.
   598→
   599→### Right way (BGM + SFX + cleanup)
   600→```js
   601→preload() {
   602→  this.load.audio('bgm', ['bgm.ogg', 'bgm.mp3']);
   603→  this.load.audio('jump', ['jump.ogg', 'jump.mp3']);
   604→}
   605→create() {
   606→  this.bgm = this.sound.add('bgm', { loop: true, volume: 0.4 });
   607→  this.bgm.play();
   608→  this.events.once('shutdown', () => this.bgm?.stop());   // CRITICAL
   609→}
   610→jump() { this.sound.play('jump'); }  // fire-and-forget (auto-destroys)
   611→```
   612→
   613→### Pitfalls
   614→- **DO NOT** use `this.sound.play(key)` if you need to control the sound later — it auto-destroys. Use `this.sound.add(key)` for a retained reference.
   615→- **DO NOT** rely on `pan` on iOS/Safari — `StereoPannerNode` not supported, events fire but no audible effect.
   616→- **DO NOT** forget `pauseOnBlur` (default `true`) pauses all sounds when tab loses focus.
   617→- **DO NOT** assume `seek` works on a stopped sound — no effect.
   618→- SoundManager is shared/global — `this.sound` in every scene references the same manager.
   619→- iOS 17/18+ can interrupt audio on background — Phaser handles via `context.suspend()`/`resume()` on `VISIBLE` game event.
   620→
   621→### v3→v4
   622→- No major changes.

   626→## 16. data-manager
   627→
   628→### Critical rules
   629→1. **Mutating a stored object/array does NOT trigger `changedata`.** The reference must change. Re-set with a new reference: `sprite.setData('inv', [...inv, 'potion'])`.
   630→2. **`values` proxy requires `set()` first.** Direct assignment `sprite.data.values.newKey = 1` does NOT create the event proxy — use `sprite.setData('newKey', 1)` first.
   631→3. **Registry listeners PERSIST across scene restarts.** Always remove them on SHUTDOWN — they are on `game.registry.events`, not `scene.events`.
   632→4. **Frozen DataManagers fail SILENTLY.** `set/remove/inc/toggle/pop/merge` all no-op without error or event. Easy to forget you froze the data.
   633→5. **`reset()` emits NO events** — clears silently. Iterate and `remove()` individually if you need removal events.
   634→
   635→### Right way (per-GO data + registry + cleanup)
   636→```js
   637→player.setData('hp', 100);
   638→player.on('changedata-hp', (go, val, prev) => { if (val <= 0) go.destroy(); });
   639→
   640→// Cross-scene via registry:
   641→this.registry.set('score', 0);
   642→// In another scene:
   643→const handler = (game, val) => this.scoreText.setText(val);
   644→this.registry.events.on('changedata-score', handler);
   645→this.events.once('shutdown', () => this.registry.events.off('changedata-score', handler));
   646→```
   647→
   648→### Pitfalls
   649→- **DO NOT** confuse `changedata` (4 args: parent, key, value, prev) with `changedata-{key}` (3 args: parent, value, prev) — key is omitted in the latter.
   650→- **DO NOT** use `inc()` on non-numeric keys — `+` operator concatenates strings, coerces booleans.
   651→- **DO NOT** expect scene data to reset on restart — `DataManagerPlugin` removes its shutdown listener but does NOT clear data. Manually `this.data.reset()` on shutdown if needed.
   652→- **DO NOT** confuse `get()` (returns copy for primitives, reference for objects) with `values` (live proxy that emits events on assignment).
   653→- Keys are case-sensitive — `'gold'` ≠ `'Gold'`.
   654→- `getAll()` returns a shallow copy — primitive values are independent, but nested objects share references.
   655→
   656→### v3→v4
   657→- No major changes.

   661→## 17. filters-and-postfx
   662→
   663→### Critical rules
   664→1. **Filters are WebGL only.** `enableFilters()` returns early if WebGL is unavailable.
   665→2. **Game objects REQUIRE `enableFilters()` before accessing `filters`.** Cameras have filters by default.
   666→3. **Internal = object-local space (cheaper, region-sized); External = screen space (full-screen, more expensive).** Use internal wherever possible.
   667→4. **Filter order matters** — output of one feeds the next.
   668→5. **Glow `quality` and `distance` are immutable** — destroy and recreate the filter to change them.
   669→
   670→### Right way (filters on sprite + camera)
   671→```js
   672→const sprite = this.add.sprite(400, 300, 'enemy');
   673→sprite.enableFilters();
   674→const glow = sprite.filters.internal.addGlow(0x00ff00, 4);
   675→glow.outerStrength = 8;        // mutable
   676→glow.setActive(false);         // toggle without removing
   677→sprite.filters.internal.remove(glow);
   678→
   679→// Camera filters (no enableFilters() needed):
   680→this.cameras.main.filters.external.addVignette(0.5, 0.5, 0.5, 0.8);
   681→
   682→// Bloom via ParallelFilters (no dedicated Bloom filter):
   683→const pf = this.cameras.main.filters.internal.addParallelFilters();
   684→pf.top.addThreshold(0.5, 1);
   685→pf.top.addBlur();
   686→pf.blend.blendMode = Phaser.BlendModes.ADD;
   687→pf.blend.amount = 0.5;
   688→```
   689→
   690→### Pitfalls
   691→- **DO NOT** use `setMask()` in v4 WebGL — logs warning. Use `filters.internal.addMask()`.
   692→- **DO NOT** forget `camera.setForceComposite(true)` for `CaptureFrame`.
   693→- **DO NOT** expect `BitmapMask` — removed in v4. Use `FilterMask`.
   694→- **DO NOT** leave many objects with active filters — each creates extra draw calls. Performance-test early.
   695→- **DO NOT** forget to set `mask.autoUpdate = false` + `mask.needsUpdate = true` for static game-object masks (default re-renders each frame).
   696→- `ignoreDestroy = true` on a controller lets it survive FilterList destruction for reuse — manage lifecycle manually.
   697→- v3 derived FX (`Bloom`, `Shine`, `Circle`, `Gradient`) removed — use Actions or new GameObjects.
   698→
   699→### v3→v4
   700→- `preFX`/`postFX` → `filters.internal`/`filters.external`.
   701→- `BitmapMask` removed → `FilterMask`.
   702→- Glow `quality` semantics changed: now an integer (default 10), not 0-1 fraction.
   703→- `enableFilters()` is a new explicit opt-in step for game objects (cameras have it by default).

   707→## 18. geometry-and-math
   708→
   709→### Critical rules
   710→1. **`Geom.Point` is REMOVED in v4.** Use `Phaser.Math.Vector2` or plain `{x, y}`. The `GEOM_CONST.POINT` constant still exists but the class doesn't.
   711→2. **Angles are in RADIANS throughout the math API.** Use `Phaser.Math.DegToRad()` / `RadToDeg()` for conversion. The `Phaser.Math.TAU` constant (2*PI) is new in v4.
   712→3. **Vector2 methods MUTATE in place** and return `this`. Call `.clone()` first to preserve the original.
   713→4. **EaseMap short names default to `.Out` variant.** `'Quad'` = `Quad.Out`, not `Quad.In`. Use full `'Quad.easeIn'`.
   714→5. **Squared distance is faster** than Euclidean (avoids `Math.sqrt`). Use `Distance.Squared` or `vec.distanceSq()` for comparisons.
   715→
   716→### Right way (vector math + intersection)
   717→```js
   718→const v = new Phaser.Math.Vector2(3, 4);
   719→v.length();      // 5
   720→const result = v.clone().normalize().scale(10);  // preserve v
   721→
   722→const rect = new Phaser.Geom.Rectangle(10, 20, 200, 100);
   723→rect.contains(50, 50);    // true
   724→Phaser.Geom.Intersects.RectangleToRectangle(rectA, rectB);
   725→
   726→const angle = Phaser.Math.Angle.Between(x1, y1, x2, y2);  // radians
   727→const clamped = Phaser.Math.Clamp(value, 0, 100);
   728→```
   729→
   730→### Pitfalls
   731→- **DO NOT** confuse `Phaser.Math.TAU` (v4 = 2*PI) with v3 (was PI/2 incorrectly). If migrating v3 code expecting PI/2, use `Phaser.Math.PI_OVER_2`.
   732→- **DO NOT** use `Math.PI2` — removed. Use `Math.TAU`.
   733→- **DO NOT** allocate Vector2 in hot loops — reuse output objects via `out` parameters.
   734→- **DO NOT** use Geom objects as game objects — they're pure data. Render via Graphics or Shape game objects.
   735→- **DO NOT** expect `Polygon.contains` to work on complex self-intersecting polygons — uses ray-casting (even/odd rule).
   736→- Intersection `Get*` functions allocate arrays — pass a reusable `out` array.
   737→
   738→### v3→v4
   739→- `Geom.Point` removed → `Vector2`.
   740→- All geometry classes return `Vector2` instead of `Point`.
   741→- `Math.TAU` value changed from `PI/2` to `PI*2`. `Math.PI2` removed. New: `Math.PI_OVER_2`.
   742→- `Phaser.Struct.Set`/`Map` → native `Set`/`Map`.

   746→## 19. graphics-and-shapes
   747→
   748→### Critical rules
   749→1. **Graphics `arc()` uses RADIANS; `this.add.arc()` Shape factory uses DEGREES.** Mixing these up is the most common bug.
   750→2. **Set style BEFORE drawing.** `fillStyle()` / `lineStyle()` must precede the corresponding `fill*` / `stroke*` call. They are NOT retroactive.
   751→3. **Graphics has NO Origin or GetBounds component** — use `setPosition(x, y)` and `displayOriginX`/`displayOriginY`. Shape game objects DO have Origin and GetBounds.
   752→4. **`generateTexture()` uses the Canvas API.** Gradient fills (`fillGradientStyle`) will NOT appear in generated textures.
   753→5. **Shape objects do NOT support `setTint()`** — use `setFillStyle(color, alpha)` / `setStrokeStyle(lineWidth, color, alpha)`.
   754→
   755→### Right way (imperative drawing vs Shape object)
   756→```js
   757→// Graphics (dynamic, multiple shapes, paths, gradients):
   758→const gfx = this.add.graphics();
   759→gfx.fillStyle(0xff0000, 1);
   760→gfx.fillRect(50, 50, 200, 100);
   761→gfx.lineStyle(3, 0x00ff00, 1);
   762→gfx.strokeCircle(400, 150, 60);
   763→
   764→// Shape object (single shape, full GO features, can be tweened/physics-enabled):
   765→const rect = this.add.rectangle(150, 100, 200, 100, 0x00aa00);
   766→rect.setStrokeStyle(3, 0xff0000);  // isStroked defaults to false — must call explicitly
   767→```
   768→
   769→### Pitfalls
   770→- **DO NOT** expect `Shape` objects to support tint methods — they don't.
   771→- **DO NOT** expect `Graphics` to have origin — it doesn't. Position via `setPosition`.
   772→- **DO NOT** forget `setStrokeStyle()` on Shapes — `isStroked` defaults to `false`.
   773→- **DO NOT** use `Graphics` for static shapes — `generateTexture()` + Sprite is faster. Graphics replays the command buffer and rebuilds geometry each frame.
   774→- **DO NOT** use `IsoBox`/`IsoTriangle` with strokes — fill-only.
   775→- **DO NOT** use `Line` shape with fill — stroke-only (constructor takes `strokeColor` not `fillColor`).
   776→- v4: Grid uses `stroke` properties (renamed from `outline` in v3). Rectangle supports rounded corners.
   777→- `pathDetailThreshold` (v4 new) skips nearby vertices in Graphics WebGL paths.
   778→
   779→### v3→v4
   780→- Grid shape: `outline` properties renamed to `stroke`.
   781→- Rectangle now supports rounded corners.
   782→- New `pathDetailThreshold` config for Graphics.

   786→## 20. loading-assets
   787→
   788→### Critical rules
   789→1. **Keys are unique per type.** A second `this.load.image('bg', ...)` is silently ignored if `'bg'` already exists. Remove the old texture first to replace.
   790→2. **Forgetting `this.load.start()` outside `preload`.** If you call load methods in `create()` or later, the Loader does NOT auto-start — call `this.load.start()` manually.
   791→3. **`setPath` appends a trailing slash; setting `this.load.path` directly requires you to include it.**
   792→4. **File keys include the prefix.** `this.load.setPrefix('MENU.')` + `this.load.image('bg', ...)` → cache key is `'MENU.bg'`. Use the full key when referencing.
   793→5. **Sprite sheet ≠ atlas.** Use `spritesheet()` for fixed-size grids (referenced by index). Use `atlas()` for packed atlases with named frames.
   794→
   795→### Right way (preload + outside-preload + chain-loading)
   796→```js
   797→preload() {
   798→  this.load.image('sky', 'assets/sky.png');
   799→  this.load.spritesheet('dude', 'assets/dude.png', { frameWidth: 32, frameHeight: 48 });
   800→  this.load.audio('jump', ['assets/jump.ogg', 'assets/jump.mp3']);
   801→}
   802→
   803→// Outside preload:
   804→create() {
   805→  this.load.image('extra', 'assets/extra.png');
   806→  this.load.once('complete', () => this.add.image(400, 300, 'extra'));
   807→  this.load.start();  // CRITICAL
   808→}
   809→
   810→// Chained loads:
   811→preload() {
   812→  this.load.json('level1', 'level1.json');
   813→  this.load.on('filecomplete-json-level1', (key, type, data) => {
   814→    this.load.image(data.images);  // batch-load from JSON
   815→  });
   816→}
   817→```
   818→
   819→### Pitfalls
   820→- **DO NOT** expect `update()` to fire during `preload()` — it's paused. `preupdate`, `postupdate`, and `render` still fire.
   821→- **DO NOT** expect progress to monotonically increase — adding files mid-load can decrease it.
   822→- **DO NOT** forget `crossOrigin: 'anonymous'` when loading from another domain (especially for WebGL textures).
   823→- **DO NOT** confuse `imageLoadType: 'XHR'` (default, blob) with `'HTMLImageElement'` — the latter helps with CORS in some environments.
   824→- **DO NOT** expect "Collection of Images" tilesets to work — Tiled parser requires single image per tileset.
   825→- The `maxRetries` property (default 2) is per-file at creation; adjusting after has no effect on existing files.
   826→- Scene `pack` config loads files BEFORE `preload()` — good for progress bar assets.
   827→
   828→### v3→v4
   829→- Default dimensions changed (1024x768 from 800x600).
   830→- `loader.maxRetries` default 2 (was 0 in early v3).

   834→## 21. particles
   835→
   836→### Critical rules
   837→1. **No `ParticleEmitterManager`** — removed since v3.60. `this.add.particles()` returns a `ParticleEmitter` directly.
   838→2. **`speed` vs `speedX`/`speedY`:** setting `speed` deactivates `speedY` (radial mode). Setting `speedX`/`speedY` switches to point mode (`radial: false`).
   839→3. **`color` overrides `tint`** — mutually exclusive; `color` (array) takes priority.
   840→4. **`frequency: 0` means emit every frame** (max rate), NOT "never." Use `emitting: false` to prevent emission.
   841→5. **`frequency: -1` = explode mode** — won't flow automatically. Use `explode()` for bursts.
   842→
   843→### Right way (flow + burst + zones)
   844→```js
   845→// Flow emitter
   846→const emitter = this.add.particles(400, 300, 'spark', {
   847→  speed: 100, lifespan: 2000,
   848→  scale: { start: 1, end: 0, ease: 'cubic.out' },
   849→  alpha: { start: 1, end: 0 },
   850→  emitZone: { type: 'random', source: new Phaser.Geom.Circle(0, 0, 50) },
   851→});
   852→
   853→// Burst
   854→const burst = this.add.particles(400, 300, 'spark', {
   855→  speed: { min: 100, max: 300 }, lifespan: 1000, emitting: false,
   856→});
   857→burst.explode(30);
   858→
   859→// Death zone (kill particles leaving a circle)
   860→emitter.addDeathZone({ type: 'onLeave', source: new Phaser.Geom.Circle(400, 300, 200) });
   861→
   862→// Follow target
   863→emitter.startFollow(player, 0, -20, true);  // x, y offset, trackVisible
   864→```
   865→
   866→### Pitfalls
   867→- **DO NOT** set both `moveToX` and `moveToY` partially — both must be set to activate; overrides `angle` and `speed`.
   868→- **DO NOT** confuse `emitting: false` (no new particles, alive ones update) with `active: false` (entire emitter frozen).
   869→- **DO NOT** confuse `'stop'` event (emission stopped) with `'complete'` (last alive particle died).
   870→- **DO NOT** expect `frequency: 0` to mean "never" — it means every frame.
   871→- **DO NOT** set `maxParticles` thinking it limits alive count — it limits TOTAL objects. Use `maxAliveParticles` for visible limit.
   872→- `hold` freezes a particle at end of life for N ms before dying.
   873→- `advance` pre-warms the emitter so particles are visible on first frame.
   874→- `reserve(count)` pre-allocates particles to avoid GC spikes.
   875→
   876→### v3→v4
   877→- No major API changes since 3.60.

   881→## 22. physics-arcade
   882→
   883→### Critical rules
   884→1. **Static bodies do NOT auto-sync.** After changing position, scale, or origin of a static body's Game Object, call `body.reset()` or `gameObject.refreshBody()`.
   885→2. **World events require opt-in.** `'collide'`, `'overlap'`, `'worldbounds'` only fire if `body.onCollide` / `body.onOverlap` / `body.onWorldBounds` is `true`.
   886→3. **`collide` separates bodies; `overlap` only detects.** Use `overlap` for triggers/pickups.
   887→4. **Persistent Colliders (`physics.add.collider`) checked every frame; one-shot `physics.collide()` must be called in `update()` each frame.**
   888→5. **Collision categories: 32 max.** Default category `0x0001`, mask `1`. PhysicsGroup defaults to mask `2147483647`. After changing categories, call `resetCollisionCategory()` to set mask to all bits.
   889→
   890→### Right way (static body + categories + collider)
   891→```js
   892→this.platforms = this.physics.add.staticGroup();
   893→this.platforms.create(400, 568, 'ground').setScale(2).refreshBody();  // CRITICAL: refreshBody
   894→
   895→const CAT_PLAYER = this.physics.nextCategory();
   896→const CAT_ENEMY  = this.physics.nextCategory();
   897→player.setCollisionCategory(CAT_PLAYER);
   898→enemy.setCollisionCategory(CAT_ENEMY);
   899→player.setCollidesWith([CAT_ENEMY]);
   900→
   901→this.physics.add.collider(player, platforms);
   902→this.physics.add.overlap(player, coins, collectCoin, null, this);
   903→
   904→// World bounds event
   905→player.body.onWorldBounds = true;
   906→this.physics.world.on('worldbounds', (body, up, down, left, right) => { /* ... */ });
   907→```
   908→
   909→### Pitfalls
   910→- **DO NOT** leave `debug: true` in production — expensive.
   911→- **DO NOT** use `useDamping: true` with large drag values — drag becomes a multiplier (e.g., `0.05` = keeps 5%/sec).
   912→- **DO NOT** confuse `immovable` (never moved by collisions) with `pushable = false` (reflects velocity to collider but still separates).
   913→- **DO NOT** change PhysicsGroup `defaults` expecting existing members to update — defaults apply only at member creation.
   914→- **DO NOT** use `overlapOnly` with TilemapLayer expecting per-tile collision settings — every tile is checked.
   915→- **DO NOT** forget `customUpdate: true` requires manual `this.physics.world.update(time, delta)` calls.
   916→- RTree (`useTree: true` default) becomes expensive to rebuild with 5000+ dynamic bodies — consider `useTree: false`.
   917→
   918→### v3→v4
   919→- No major changes.

   923→## 23. render-textures
   924→
   925→### Critical rules
   926→1. **`render()` is REQUIRED.** v4 buffers all drawing commands; nothing appears until `render()` is called (unless `renderMode: 'all'` or `'redraw'`).
   927→2. **Command buffer CLEARS after `render()` by default.** Call `preserve(true)` to keep commands for replay.
   928→3. **`stamp()` ignores the internal camera** — only `draw()` and `capture()` respect the `.camera` property. Texture stamps are positioned absolutely.
   929→4. **`resize()` erases content** — destroys and recreates the framebuffer.
   930→5. **`saveTexture(key)` RENAMES, does not copy.** Calling again with a different key renames. Destroying the RenderTexture without saving first destroys the texture.
   931→
   932→### Right way (RT + DynamicTexture + minimap)
   933→```js
   934→// RenderTexture (visible game object)
   935→const rt = this.add.renderTexture(0, 0, 256, 256).setOrigin(0);
   936→rt.clear();
   937→rt.fill(0x000000, 0.5);
   938→rt.draw(sprite, 128, 128);
   939→rt.render();  // CRITICAL
   940→
   941→// DynamicTexture (shared texture, cross-scene)
   942→const dt = this.textures.addDynamicTexture('composite', 512, 512);
   943→dt.stamp('coin', null, 64, 64, { scale: 2, angle: 45 });
   944→dt.render();
   945→this.add.image(400, 300, 'composite');
   946→
   947→// Auto-render minimap
   948→this.minimap = this.add.renderTexture(700, 50, 150, 100);
   949→this.minimap.setScrollFactor(0).setRenderMode('all').preserve(true);
   950→```
   951→
   952→### Pitfalls
   953→- **DO NOT** draw a DynamicTexture to itself — silently skips.
   954→- **DO NOT** use `alpha`/`tint` params on `draw()` with game objects — they only apply to Texture Frames/strings. Game objects use their own alpha/tint.
   955→- **DO NOT** snapshot every frame — `readPixels` blocks the GPU pipeline. Use `snapshotPixel` (cheaper) or cache.
   956→- **DO NOT** expect anti-aliasing on WebGL1 framebuffers — jagged edges. Pre-render sprites instead.
   957→- **DO NOT** forget to handle WebGL context loss — DynamicTexture contents are lost. Listen for `restorewebgl`.
   958→- Default sizes differ: RenderTexture 32x32, DynamicTexture 256x256.
   959→- `forceEven` rounds dimensions up by default — pass `false` for exact odd dimensions.
   960→
   961→### v3→v4
   962→- **BREAKING:** Drawing commands are now BUFFERED. Must call `.render()` explicitly. In v3, draw commands executed immediately.
   963→- New: `preserve()`, `callback()`, `capture()`, `renderMode` ('render'/'redraw'/'all').
   964→- `TextureManager.addDynamicTexture` now has `forceEven` parameter.

   968→## 24. scale-and-responsive
   969→
   970→### Critical rules
   971→1. **Parent element must have calculable dimensions.** Unstyled `<div>` has zero height — breaks centering and scaling. Use `expandParent: true` (default) or set explicit CSS.
   972→2. **Do NOT style the canvas directly.** ScaleManager controls `canvas.style.width/height/marginLeft/marginTop`. External CSS conflicts.
   973→3. **Fullscreen MUST be triggered from `pointerup` (not `pointerdown`)** — touch devices block `pointerdown` requests.
   974→4. **`resize()` is for NONE mode only; `setGameSize()` for FIT/ENVELOP/etc.** Mixing them up breaks scaling.
   975→5. **RESIZE mode = 1:1 pixel mapping** — large displays cause GPU fill-rate pressure on low-end devices.
   976→
   977→### Right way (FIT + center + fullscreen)
   978→```js
   979→const config = {
   980→  scale: {
   981→    parent: 'game-container',
   982→    mode: Phaser.Scale.FIT,
   983→    autoCenter: Phaser.Scale.CENTER_BOTH,
   984→    width: 1280, height: 720,
   985→    min: { width: 640, height: 360 },
   986→    max: { width: 1920, height: 1080 },
   987→  },
   988→  scene: MyScene,
   989→};
   990→
   991→// In scene:
   992→this.scale.on('resize', (gameSize, baseSize, displaySize) => {
   993→  this.cameras.resize(gameSize.width, gameSize.height);
   994→  // Reposition UI
   995→});
   996→
   997→// Fullscreen (pointerup, not pointerdown):
   998→this.input.on('pointerup', () => this.scale.toggleFullscreen());
   999→```
  1000→
  1001→### Pitfalls
  1002→- **DO NOT** apply padding to the parent — ScaleManager doesn't account for it. Use a wrapper element.
  1003→- **DO NOT** use `resize()` in FIT mode — use `setGameSize()`.
  1004→- **DO NOT** expect fullscreen to work in iframes without `allowfullscreen` attribute.
  1005→- **DO NOT** set `width: '100%'` without a sized parent — falls back to `window.innerWidth/innerHeight`.
  1006→- `resizeInterval` (default 500ms) is a fallback poll on top of browser resize events.
  1007→- iOS Safari height quirks handled via `GetInnerHeight` workaround.
  1008→
  1009→### v3→v4
  1010→- New `Phaser.Scale.EXPAND` mode (value 6) — hybrid of RESIZE + FIT.
  1011→

  1014→## 25. sprites-and-images
  1015→
  1016→### Critical rules
  1017→1. **Use `Image` instead of `Sprite` when you don't need animation.** Sprites are added to the update list and run `preUpdate` every frame. Images skip this overhead.
  1018→2. **`z` does NOT control render order.** Use `depth` (Depth component) or `setToTop()`/`setAbove()` etc.
  1019→3. **`scale` getter returns the AVERAGE of `scaleX` and `scaleY`.** Check `scaleX`/`scaleY` individually when non-uniform.
  1020→4. **`setFrame()` resets size and origin by default.** Pass `false, false` to prevent: `setFrame('f', false, false)`.
  1021→5. **Setting `alpha = 0` or `scale = 0` clears a render flag** — the object is NOT drawn at all. Use `setVisible(false)` if you want to explicitly hide without changing alpha.
  1022→
  1023→### Right way (chained setup)
  1024→```js
  1025→const player = this.add.sprite(100, 200, 'player', 'idle-0')
  1026→  .setScale(2)
  1027→  .setOrigin(0.5, 1)        // bottom-center
  1028→  .setDepth(10)
  1029→  .setFlip(true, false)
  1030→  .setAlpha(0.8)
  1031→  .play('idle');
  1032→
  1033→// Change frame without resetting size/origin:
  1034→player.setFrame('walk-1', false, false);
  1035→```
  1036→
  1037→### Pitfalls
  1038→- **DO NOT** use `setTintFill()` — REMOVED in v4. Use `setTint(color).setTintMode(Phaser.TintModes.FILL)`.
  1039→- **DO NOT** expect flip to affect physics bodies — rendering-only.
  1040→- **DO NOT** set `scrollFactor` on physics-bodied objects expecting collisions to follow — physics uses world position; scroll factor offsets rendering only.
  1041→- **DO NOT** use Sprite for static decor — Image is cheaper.
  1042→- **DO NOT** assume `rotation` is in degrees — it's radians. Use `angle` for degrees.
  1043→- Rotation direction: right-hand clockwise (0=right, 90°=down, 180°=left, -90°=up).
  1044→- `ignoreDestroy = true` prevents destruction — you manage lifecycle.
  1045→- `isDestroyed` boolean tracks destroy() state.
  1046→
  1047→### v3→v4
  1048→- `setTintFill()` removed.
  1049→- Tint color and tint mode are now separate properties.
  1050→- TileSprite supports atlas frames and `tileRotation` (new in v4).
  1051→- TileSprite no longer supports texture cropping.

  1055→## 26. tilemaps
  1056→
  1057→### Critical rules
  1058→1. **Tileset name in `addTilesetImage` must match Tiled EXACTLY** — first arg is the Tiled tileset name, NOT the Phaser texture key. If they differ, you get `null` and a console warning.
  1059→2. **Layer name in `createLayer` must match Tiled EXACTLY.** Group layer children are prefixed `'GroupName/LayerName'`.
  1060→3. **Each layer can only be created ONCE.** Calling `createLayer` with the same layer ID twice returns `null`.
  1061→4. **`setCollision` MUST be called before physics colliders work.** Without marking tiles as collidable, `physics.add.collider()` passes through all tiles.
  1062→5. **TilemapGPULayer requires `generateLayerDataTexture()` after tile edits** — changes won't appear otherwise. Orthographic only; single tileset per layer.
  1063→
  1064→### Right way (Tiled JSON + collision + physics)
  1065→```js
  1066→preload() {
  1067→  this.load.tilemapTiledJSON('map', 'assets/map.json');
  1068→  this.load.image('tiles', 'assets/tileset.png');
  1069→}
  1070→create() {
  1071→  const map = this.add.tilemap('map');
  1072→  const tileset = map.addTilesetImage('TilesetNameInTiled', 'tiles');  // CRITICAL: Tiled name first
  1073→  const ground = map.createLayer('Ground', tileset);                   // CRITICAL: layer name from Tiled
  1074→  ground.setCollisionByProperty({ collides: true });                    // CRITICAL: before physics
  1075→  this.physics.add.collider(player, ground);
  1076→
  1077→  // Object layer → sprites
  1078→  const coins = map.createFromObjects('Items', { name: 'coin', key: 'coin-texture' });
  1079→}
  1080→```
  1081→
  1082→### Pitfalls
  1083→- **DO NOT** use "Collection of Images" tilesets — Tiled parser requires single image per tileset. Embedded tilesets required.
  1084→- **DO NOT** expect `setTileIndexCallback` to fire without an active physics collider/overlap.
  1085→- **DO NOT** pass `x, y` to `createLayer` thinking they default to (0,0) — they default to the layer offset defined in Tiled.
  1086→- **DO NOT** use `TilemapGPULayer` for isometric/hex/staggered maps — orthographic only.
  1087→- **DO NOT** use multiple tilesets on a single `TilemapGPULayer` — single tileset only.
  1088→- Tile index `-1` means empty; methods return `null` by default. Pass `nonNull: true` to get a Tile with index -1.
  1089→- `insertNull: true` in tilemap factory stores `null` for empty cells (saves memory, prevents dynamic placement in empty cells).
  1090→- `TilemapGPULayer` max 4096x4096 tiles, up to 2^23 unique tile IDs.
  1091→
  1092→### v3→v4
  1093→- New `TilemapGPULayer` (WebGL only, orthographic, single tileset, single quad render).
  1094→- Tilemap now supports animated tiles on both CPU and GPU layers.
  1095→- `createLayer` 5th arg `gpu: true` enables GPU layer.

  1099→## 27. curves-and-paths
  1100→
  1101→### Critical rules
  1102→1. **`getPoint(t)` uses raw parameter; `getPointAt(u)` uses arc length (evenly spaced).** On a Path, `getPoint` already accounts for arc length across the whole path — but on individual curves, use `getPointAt` for even spacing.
  1103→2. **PathFollower uses a Tween internally.** All tween properties (`ease`, `delay`, `repeat`, `yoyo`, callbacks) work. The tween is `persist: true` automatically.
  1104→3. **`positionOnPath: false` (default)** — follower's current position becomes the offset from path start. With `true`, the follower snaps to path start and offset is zeroed.
  1105→4. **Ellipse curve angles are in DEGREES** — constructor and `startAngle`/`endAngle` accept degrees. Internally stored as radians. `rotation` is radians; `angle` is degrees.
  1106→5. **`cubicBezierTo` parameter order with numbers is (endX, endY, cp1X, cp1Y, cp2X, cp2Y)** — end point FIRST, not control points. With Vector2 objects, it's `(cp1, cp2, endPoint)`.
  1107→
  1108→### Right way (path + follower)
  1109→```js
  1110→const path = this.add.path(100, 200);
  1111→path.lineTo(400, 400);
  1112→path.splineTo([new Phaser.Math.Vector2(600, 300), new Phaser.Math.Vector2(700, 200)]);
  1113→
  1114→const enemy = this.add.follower(path, 100, 200, 'enemy');
  1115→enemy.startFollow({
  1116→  duration: 3000,
  1117→  positionOnPath: true,    // snap to path start
  1118→  rotateToPath: true,      // auto-rotate to face direction
  1119→  rotationOffset: 90,      // degrees
  1120→  repeat: -1,
  1121→  yoyo: true,
  1122→});
  1123→
  1124→// Draw the path:
  1125→const gfx = this.add.graphics();
  1126→gfx.lineStyle(2, 0xffffff, 1);
  1127→path.draw(gfx, 64);  // 64 points per curve
  1128→```
  1129→
  1130→### Pitfalls
  1131→- **DO NOT** expect `moveTo()` to draw — it creates an inactive pseudo-curve that only repositions the end point.
  1132→- **DO NOT** expect cached lengths to update when modifying curve control points — call `path.updateArcLengths()`.
  1133→- **DO NOT** confuse `closePath()` (adds an explicit Line curve) with `autoClose = true` (only affects `getPoints` output).
  1134→- **DO NOT** use Spline with fewer than 4 points — Catmull-Rom interpolation works best with 4+.
  1135→- Line curve overrides `arcLengthDivisions` to 1 (uniform by nature).
  1136→
  1137→### v3→v4
  1138→- No major changes.

  1142→## 28. actions-and-utilities
  1143→
  1144→### Critical rules
  1145→1. **Actions operate on PLAIN ARRAYS, not Groups directly.** Always call `group.getChildren()` first.
  1146→2. **`PlaceOn*`/`Random*` actions need `Phaser.Geom` objects**, not Game Object shapes. For a `Phaser.GameObjects.Circle`, pass `circle.geom`.
  1147→3. **`SetXY` defaults `y` to `x`** if `y` is `undefined` or `null`. Pass `0` explicitly if you want y=0 and x=something else.
  1148→4. **`step` is multiplied by iteration index, not added cumulatively.** Item 0 gets `value + 0*step`, item 1 gets `value + 1*step`, etc.
  1149→5. **`Spread` with a single item places it at the midpoint `(min+max)/2`, NOT at `min`.**
  1150→
  1151→### Right way (grid align + scatter + wrap)
  1152→```js
  1153→const sprites = [];
  1154→for (let i = 0; i < 20; i++) sprites.push(this.add.sprite(0, 0, 'gem'));
  1155→
  1156→Phaser.Actions.GridAlign(sprites, {
  1157→  width: 5, height: 4, cellWidth: 64, cellHeight: 64, x: 100, y: 100,
  1158→});
  1159→
  1160→// With a Group:
  1161→const enemies = this.add.group({ key: 'enemy', repeat: 9 });
  1162→Phaser.Actions.PlaceOnCircle(enemies.getChildren(), new Phaser.Geom.Circle(400, 300, 200));
  1163→
  1164→// Step parameter for staggered values:
  1165→Phaser.Actions.SetX(sprites, 100, 50);  // x: 100, 150, 200, 250...
  1166→```
  1167→
  1168→### Pitfalls
  1169→- **DO NOT** pass a Group to Actions — they expect arrays. Use `group.getChildren()`.
  1170→- **DO NOT** pass a `Phaser.GameObjects.Circle` to `PlaceOnCircle` — pass `circle.geom` (a `Phaser.Geom.Circle`).
  1171→- **DO NOT** use `Shuffle` expecting a new array — it modifies in place and returns the same array.
  1172→- **DO NOT** use `GetFastValue` with a dot-path key — it's top-level only. Use `GetValue` for dot paths.
  1173→- `AddEffectBloom` / `AddEffectShine` / `AddMaskShape` are v4-only filter-based effects — they return arrays of created effects, not the input array.
  1174→
  1175→### v3→v4
  1176→- New v4-only effect actions: `AddEffectBloom`, `AddEffectShine`, `AddMaskShape`.
  1177→- New `FitToRegion` action.

  1182→
  1183→Use this consolidated checklist when auditing any Phaser 4.2.1 game:
  1184→
  1185→### Listener cleanup (most common leak)
  1186→- [ ] Every `this.events.on(...)` has a matching `off()` in a `shutdown` listener (or uses `once()`).
  1187→- [ ] Every `this.input.on(...)` has a matching `off()` in `shutdown`.
  1188→- [ ] Every `this.game.events.on(...)` has a matching `off()` in `shutdown` (persists across restarts!).
  1189→- [ ] Every `this.registry.events.on('changedata-*', handler)` has a matching `off()` in `shutdown`.
