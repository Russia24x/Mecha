     8‚Üí     8‚Üí## 1. physics-matter ‚ö° (HIGH-PRIORITY AUDIT TARGET)
     9‚Üí     9‚Üí
    10‚Üí    10‚Üí### Critical rules
    11‚Üí    11‚Üí1. **Matter body position = center of mass**, not top-left (unlike Arcade). `this.matter.add.sprite(x, y, 't')` places the body's center at `(x, y)`.
    12‚Üí    12‚Üí2. **`setBody` / `setRectangle` / `setCircle` / `setPolygon` RESET ALL PROPERTIES** ‚Äî mass, friction, collision filters, callbacks are all wiped. Re-apply after any shape change.
    13‚Üí    13‚Üí3. **Force values are tiny** (`0.01`‚Äì`0.1`); velocity values are `1`‚Äì`15`. They are NOT pixel-based. `applyForce({x: 0.05, y: 0})` is correct; `applyForce({x: 50, y: 0})` will rocket the body.
    14‚Üí    14‚Üí4. **`collisionFilter.group` overrides category/mask.** Same positive group = always collide; same negative group = never collide; `0`/different = use category/mask. Sensors still require matching filters to fire events.
    15‚Üí    15‚Üí5. **Constraints must target the parent body**, never compound body `parts`. 32 collision categories max.
    16‚Üí    16‚Üí
    17‚Üí    17‚Üí### Right way (compound body + collision callback)
    18‚Üí    18‚Üí```js
    19‚Üí    19‚Üíconst compound = this.matter.body.create({ parts: [partA, partB] });
    20‚Üí    20‚Üíconst player = this.matter.add.sprite(400, 200, 'hero');
    21‚Üí    21‚Üíplayer.setExistingBody(compound);          // re-apply mass/friction/etc. after this
    22‚Üí    22‚Üíplayer.setOnCollide((pair) => { /* pair.bodyA, pair.bodyB */ });
    23‚Üí    23‚Üí
    24‚Üí    24‚Üí// Collision categories
    25‚Üí    25‚Üíconst PLAYER = this.matter.world.nextCategory();  // 0x0002
    26‚Üí    26‚Üíconst ENEMY  = this.matter.world.nextCategory();
    27‚Üí    27‚Üíplayer.setCollisionCategory(PLAYER);
    28‚Üí    28‚Üíplayer.setCollidesWith([ENEMY]);
    29‚Üí    29‚Üí```
    30‚Üí    30‚Üí
    31‚Üí    31‚Üí### Pitfalls
    32‚Üí    32‚Üí- **DO NOT** call `setBody()` and assume collision filters persist ‚Äî they're reset.
    33‚Üí    33‚Üí- **DO NOT** target `compound.parts[i]` in `add.constraint()` ‚Äî use the parent.
    34‚Üí    34‚Üí- **DO NOT** forget `setSleepEvents(true, true)` if you want `sleepstart`/`sleepend` to fire.
    35‚Üí    35‚Üí- **DO NOT** call `convertTilemapLayer(layer)` before `layer.setCollisionByProperty(...)` ‚Äî collision must be set first.
    36‚Üí    36‚Üí- **DO NOT** expect `restitution` to average ‚Äî `Math.max(bodyA.restitution, bodyB.restitution)` wins.
    37‚Üí    37‚Üí- `body.ignorePointer = true` prevents the mouseSpring from dragging that body.
    38‚Üí    38‚Üí
    39‚Üí    39‚Üí### v3‚Üív4
    40‚Üí    40‚Üí- No major Matter API change; `MatterGameObject` injection via `this.matter.add.gameObject(go, opts)` still works.

    44‚Üí    44‚Üí## 2. input-keyboard-mouse-touch ‚ö° (HIGH-PRIORITY AUDIT TARGET)
    45‚Üí    45‚Üí
    46‚Üí    46‚Üí### Critical rules
    47‚Üí    47‚Üí1. **`setInteractive()` MUST be called before listening for Game Object pointer events.** Without it, `sprite.on('pointerdown', ...)` will never fire.
    48‚Üí    48‚Üí2. **Pointer events fire at 3 levels** ‚Äî Game Object (`pointerdown`), Scene per-object (`gameobjectdown`), Scene global (`pointerdown`). Higher handlers can `event.stopPropagation()`.
    49‚Üí    49‚Üí3. **`pointer.worldX/worldY` are only valid inside input handlers.** Outside, call `pointer.updateWorldPoint(camera)` first.
    50‚Üí    50‚Üí4. **`topOnly` defaults to `true`** ‚Äî only the top-most interactive object receives events. Set `this.input.topOnly = false` to receive on all objects under the pointer.
    51‚Üí    51‚Üí5. **Gamepad callbacks differ between plugin and instance.** Plugin-level: `(pad, button, value)`. Per-pad: `(buttonIndex, value, button)`. Mixing these is a classic bug.
    52‚Üí    52‚Üí
    53‚Üí    53‚Üí### Right way (interactive sprite + keyboard)
    54‚Üí    54‚Üí```js
    55‚Üí    55‚Üísprite.setInteractive({ useHandCursor: true });
    56‚Üí    56‚Üísprite.on('pointerdown', (pointer, localX, localY, event) => {
    57‚Üí    57‚Üí  // localX/localY are relative to the GO's top-left
    58‚Üí    58‚Üí  event.stopPropagation();
    59‚Üí    59‚Üí});
    60‚Üí    60‚Üí
    61‚Üí    61‚Üí// Keyboard with capture (prevents browser scroll on arrow keys)
    62‚Üí    62‚Üíthis.input.keyboard.addCapture('SPACE,UP,DOWN,LEFT,RIGHT');
    63‚Üí    63‚Üíconst space = this.input.keyboard.addKey('SPACE');
    64‚Üí    64‚Üí// One-shot detection
    65‚Üí    65‚Üíif (Phaser.Input.Keyboard.JustDown(space)) { /* fire once */ }
    66‚Üí    66‚Üí```
    67‚Üí    67‚Üí
    68‚Üí    68‚Üí### Pitfalls
    69‚Üí    69‚Üí- **DO NOT** use `pixelPerfect: true` for many objects ‚Äî expensive (per-pixel alpha test). Prefer geometric hit areas.
    70‚Üí    70‚Üí- **DO NOT** forget `this.input.setDraggable(sprite)` (or `{draggable: true}` in `setInteractive`) ‚Äî `setInteractive()` alone does NOT enable drag.
    71‚Üí    71‚Üí- **DO NOT** assume `pointer.x/y` are world coords ‚Äî they are screen coords. Use `pointer.worldX/Y` or `cam.getWorldPoint()`.
    72‚Üí    72‚Üí- **DO NOT** assume gamepad is connected on scene start ‚Äî check `this.input.gamepad.total > 0` and use `pad1`‚Äì`pad4`. `'connected'` only fires for NEW connections.
    73‚Üí    73‚Üí- **DO NOT** expect touch events during `preload()` ‚Äî they are not dispatched until `create`.
    74‚Üí    74‚Üí- Browser extensions (Vimium) can intercept keys ‚Äî verify when debugging.
    75‚Üí    75‚Üí- Gamepad API requires HTTPS in modern browsers.
    76‚Üí    76‚Üí
    77‚Üí    77‚Üí### v3‚Üív4
    78‚Üí    78‚Üí- No major changes; `Button` class accepts `isPressed` parameter for cross-scene-transition state correctness.
    79‚Üí    79‚Üí

    82‚Üí    82‚Üí## 3. text-and-bitmaptext ‚ö° (HIGH-PRIORITY AUDIT TARGET)
    83‚Üí    83‚Üí
    84‚Üí    84‚Üí### Critical rules
    85‚Üí    85‚Üí1. **Canvas `Text` re-renders the entire texture on every `setText`/`setStyle`/`setColor`/`setFontSize`.** Batch changes and call `updateText()` once. For frequently-updated text (scores, timers, damage numbers), use **BitmapText**.
    86‚Üí    86‚Üí2. **Text origin defaults to (0,0)** ‚Äî top-left. Sprites default to (0.5, 0.5). Call `text.setOrigin(0.5)` to center.
    87‚Üí    87‚Üí3. **`setRTL(true)` MUST be called BEFORE `setText()`** ‚Äî RTL changes how the canvas is constructed.
    88‚Üí    88‚Üí4. **BitmapText font must be in cache** ‚Äî load via `this.load.bitmapFont(key, png, xml/json)`. Missing font = console warning + undefined behavior.
    89‚Üí    89‚Üí5. **`setCharacterTint`/`setWordTint`/`setDropShadow` are WebGL only** ‚Äî no effect in Canvas renderer.
    90‚Üí    90‚Üí
    91‚Üí    91‚Üí### Right way (frequently updated text)
    92‚Üí    92‚Üí```js
    93‚Üí    93‚Üí// BAD for score updates each frame:
    94‚Üí    94‚ÜíscoreText.setText('Score: ' + score);  // re-creates canvas + re-uploads GPU texture
    95‚Üí    95‚Üí
    96‚Üí    96‚Üí// GOOD: use BitmapText
    97‚Üí    97‚Üíthis.load.bitmapFont('pixelFont', 'font.png', 'font.xml');
    98‚Üí    98‚Üíconst scoreText = this.add.bitmapText(10, 10, 'pixelFont', 'Score: 0', 24);
    99‚Üí    99‚ÜíscoreText.setText('Score: ' + score);  // only updates vertex data
   100‚Üí   100‚Üí```
   101‚Üí   101‚Üí
   102‚Üí   102‚Üí### Pitfalls
   103‚Üí   103‚Üí- **DO NOT** call `setColor()` thinking it will only change color ‚Äî it triggers a full canvas rebuild.
   104‚Üí   104‚Üí- **DO NOT** expect `align: 'center'` to center single-line text ‚Äî it only affects multi-line. Use `setOrigin(0.5)` for single-line centering.
   105‚Üí   105‚Üí- **DO NOT** use non-zero `letterSpacing` on Text for large bodies of text ‚Äî each character is rendered individually. Use BitmapText instead.
   106‚Üí   106‚Üí- **DO NOT** use `setDropShadow` on DynamicBitmapText or in Canvas renderer ‚Äî no effect.
   107‚Üí   107‚Üí- **DO NOT** forget to quote font names with digits/special chars: `fontFamily: '"Press Start 2P"'`.
   108‚Üí   108‚Üí- `setFill()` and `setColor()` are equivalent aliases.
   109‚Üí   109‚Üí
   110‚Üí   110‚Üí### v3‚Üív4
   111‚Üí   111‚Üí- No breaking changes; BitmapText tinting now works correctly in v4.
   112‚Üí   112‚Üí
   113‚Üí   113‚Üí---

   115‚Üí   115‚Üí## 4. groups-and-containers ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   116‚Üí   116‚Üí
   117‚Üí   117‚Üí### Critical rules
   118‚Üí   118‚Üí1. **Container is EXCLUSIVE by default** ‚Äî adding a child to a new Container removes it from its previous one. Use `container.setExclusive(false)` to override.
   119‚Üí   119‚Üí2. **`Container.setScrollFactor(x, y, updateChildren)` does NOT propagate to children by default.** Pass `true` as the third arg: `container.setScrollFactor(0, 0, true)`.
   120‚Üí   120‚Üí3. **Group is NOT on the display list** ‚Äî moving a Group does nothing visually. Use Container for transform inheritance.
   121‚Üí   121‚Üí4. **Container origin is always (0,0)** ‚Äî position children relative to (0,0); the origin cannot be changed.
   122‚Üí   122‚Üí5. **Container needs `setSize(w, h)` before `setInteractive()` works** ‚Äî containers have no implicit size. Pixel-perfect hit testing does NOT work on Containers.
   123‚Üí   123‚Üí
   124‚Üí   124‚Üí### Right way (HUD pinned to camera)
   125‚Üí   125‚Üí```js
   126‚Üí   126‚Üíconst hud = this.add.container(10, 10);
   127‚Üí   127‚Üíhud.setScrollFactor(0, 0, true);  // CRITICAL: third arg = true propagates to children
   128‚Üí   128‚Üíhud.add([healthBar, scoreText, livesIcon]);
   129‚Üí   129‚Üí```
   130‚Üí   130‚Üí
   131‚Üí   131‚Üí### Pitfalls
   132‚Üí   132‚Üí- **DO NOT** put physics bodies on Container children unless the Container is at (0,0) ‚Äî bodies will be offset.
   133‚Üí   133‚Üí- **DO NOT** mix `depth` and display-list reordering ‚Äî `depth` takes priority after the next sort pass.
   134‚Üí   134‚Üí- **DO NOT** expect `group.add(child)` to put the child on the display list ‚Äî only `group.create()` (with `true` second arg) or `this.add.existing()` does.
   135‚Üí   135‚Üí- **DO NOT** mask Container children individually in Canvas ‚Äî only the Container itself can be masked (WebGL: masks stack).
   136‚Üí   136‚Üí- **DO NOT** put a Layer inside a Container ‚Äî Layer cannot go in Container (reverse is OK).
   137‚Üí   137‚Üí- `killAndHide(obj)` does NOT remove from group ‚Äî it only sets `active=false, visible=false` for reuse.
   138‚Üí   138‚Üí- Container has matrix-math per child per frame ‚Äî deep nesting multiplies cost.
   139‚Üí   139‚Üí
   140‚Üí   140‚Üí### v3‚Üív4
   141‚Üí   141‚Üí- No major changes. Layer is the preferred render-bucketing primitive; Container for transform inheritance.
   142‚Üí   142‚Üí
   143‚Üí   143‚Üí---

   145‚Üí   145‚Üí## 5. scenes ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   146‚Üí   146‚Üí
   147‚Üí   147‚Üí### Critical rules
   148‚Üí   148‚Üí1. **All `this.scene.*` operations are QUEUED, not immediate.** `this.scene.start('X')` happens on the next SceneManager update ‚Äî do not rely on X's state in the same frame.
   149‚Üí   149‚Üí2. **`start()` SHUTS DOWN the calling scene.** Use `launch()` or `run()` if you want both running in parallel.
   150‚Üí   150‚Üí3. **`switch()` sleeps the current scene; `start()` shuts it down.** `switch()` preserves state; `start()` triggers full SHUTDOWN.
   151‚Üí   151‚Üí4. **Paused scenes STILL RENDER.** Use `sleep()` to stop both update and render.
   152‚Üí   152‚Üí5. **Reset state in `init()`, NOT the constructor.** Constructor runs once per instantiation; `init()` runs every start/restart.
   153‚Üí   153‚Üí
   154‚Üí   154‚Üí### Right way (state reset + shutdown cleanup)
   155‚Üí   155‚Üí```js
   156‚Üí   156‚Üíclass GameScene extends Phaser.Scene {
   157‚Üí   157‚Üí  constructor() { super('GameScene'); }
   158‚Üí   158‚Üí  init() { this.score = 0; this.gameOver = false; }  // resets every restart
   159‚Üí   159‚Üí  create() {
   160‚Üí   160‚Üí    // CRITICAL: clean up on shutdown to avoid stale refs on restart
   161‚Üí   161‚Üí    this.events.once('shutdown', () => { this.enemies = []; });
   162‚Üí   162‚Üí  }
   163‚Üí   163‚Üí}
   164‚Üí   164‚Üí```
   165‚Üí   165‚Üí
   166‚Üí   166‚Üí### Pitfalls
   167‚Üí   167‚Üí- **DO NOT** overwrite `this.sys` ‚Äî breaks everything.
   168‚Üí   168‚Üí- **DO NOT** assume `switch()` resumes a paused scene ‚Äî it RESTARTS. Use `run()` for smart resume/wake/start.
   169‚Üí   169‚Üí- **DO NOT** expect sleeping scenes to be silent ‚Äî they can still receive events from other scenes (e.g., global registry events).
   170‚Üí   170‚Üí- **DO NOT** rely on constructor state resets ‚Äî they only run once.
   171‚Üí   171‚Üí- **DO NOT** forget that `pause()` keeps rendering; if you want a frozen scene, use `sleep()`.
   172‚Üí   172‚Üí- Scenes update in **reverse array order**, render in **forward array order** (last scene in array renders on top).
   173‚Üí   173‚Üí- `create` event fires AFTER `create()` returns.
   174‚Üí   174‚Üí
   175‚Üí   175‚Üí### v3‚Üív4
   176‚Üí   176‚Üí- No major changes; InjectionMap customization via `map` config still supported.
   177‚Üí   177‚Üí
   178‚Üí   178‚Üí---

   180‚Üí   180‚Üí## 6. cameras ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   181‚Üí   181‚Üí
   182‚Üí   182‚Üí### Critical rules
   183‚Üí   183‚Üí1. **`this.cameras` is the CameraManager, NOT an array.** Array is `this.cameras.cameras`; main camera is `this.cameras.main`.
   184‚Üí   184‚Üí2. **Viewport vs Scroll are independent.** `setViewport/setPosition/setSize` = on-screen rectangle. `setScroll/scrollX/scrollY` = what world area the camera looks at.
   185‚Üí   185‚Üí3. **`cam.startFollow(target)` SNAPS on first call.** To avoid a visible snap, set scroll to the target's position first.
   186‚Üí   186‚Üí4. **Effects do NOT restart by default.** Calling `cam.fade()` while a fade is running does nothing unless you pass `force: true`.
   187‚Üí   187‚Üí5. **`pan()` OVERRIDES follow** while running ‚Äî follow resumes after pan completes.
   188‚Üí   188‚Üí
   189‚Üí   189‚Üí### Right way (HUD camera + main camera with bounds)
   190‚Üí   190‚Üí```js
   191‚Üí   191‚Üíconst main = this.cameras.main;
   192‚Üí   192‚Üímain.setBounds(0, 0, 2048, 2048);
   193‚Üí   193‚Üímain.startFollow(player, false, 0.1, 0.1);  // lerp 0.1 = smooth
   194‚Üí   194‚Üí
   195‚Üí   195‚Üí// HUD camera (does not scroll)
   196‚Üí   196‚Üíconst hudCam = this.cameras.add(0, 0, 800, 600).setName('hud');
   197‚Üí   197‚Üímain.ignore(hudGroup);          // main cam doesn't draw HUD
   198‚Üí   198‚ÜíhudCam.ignore(worldGroup);      // HUD cam doesn't draw world
   199‚Üí   199‚Üí
   200‚Üí   200‚Üí// Convert pointer to world coords (essential with scroll/zoom)
   201‚Üí   201‚Üíconst wp = main.getWorldPoint(pointer.x, pointer.y);
   202‚Üí   202‚Üí```
   203‚Üí   203‚Üí
   204‚Üí   204‚Üí### Pitfalls
   205‚Üí   205‚Üí- **DO NOT** set zoom to 0 ‚Äî clamped to 0.001 internally but breaks rendering.
   206‚Üí   206‚Üí- **DO NOT** use `setMask()` in WebGL ‚Äî logs warning. Use `cam.filters.external.addMask()` instead. `setMask` is Canvas-only (GeometryMask).
   207‚Üí   207‚Üí- **DO NOT** use non-integer zoom with `roundPixels: true` ‚Äî causes jitter.
   208‚Üí   208‚Üí- **DO NOT** forget to call `camControl.update(delta)` in `update()` for `FixedKeyControl`/`SmoothedKeyControl` ‚Äî they do not auto-update.
   209‚Üí   209‚Üí- **DO NOT** rotate the viewport ‚Äî it's always axis-aligned; rotation is render-only.
   210‚Üí   210‚Üí- **DO NOT** create more than 32 cameras if you need `ignore()` ‚Äî cameras 33+ get ID 0 and cannot exclude objects.
   211‚Üí   211‚Üí- `setBounds` only restricts scrolling ‚Äî objects can still be placed outside.
   212‚Üí   212‚Üí
   213‚Üí   213‚Üí### v3‚Üív4
   214‚Üí   214‚Üí- **Camera matrix rewritten.** `Camera#matrix` now = rotation + zoom + scroll (no position). `matrixExternal` = position only. `matrixCombined` = `matrix * matrixExternal`. If you accessed matrices directly, update code.
   215‚Üí   215‚Üí- Use `TransformMatrix.copyWithScrollFactorFrom(matrix, scrollX, scrollY, scrollFactorX, scrollFactorY)` instead of manually subtracting `camera.scrollX * src.scrollFactorX`.
   216‚Üí   216‚Üí- Cameras have `filters.internal`/`filters.external` by default ‚Äî no `enableFilters()` needed.
   217‚Üí   217‚Üí
   218‚Üí   218‚Üí---

   220‚Üí   220‚Üí## 7. game-object-components ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   221‚Üí   221‚Üí
   222‚Üí   222‚Üí### Critical rules
   223‚Üí   223‚Üí1. **`alpha = 0`, `scale = 0`, `visible = false` ALL clear a render flag** ‚Äî the object is NOT drawn at all (not just transparent).
   224‚Üí   224‚Üí2. **`setTintFill()` is REMOVED in v4.** Use `sprite.setTint(color).setTintMode(Phaser.TintModes.FILL)`. Calling `setTintFill()` logs a console error.
   225‚Üí   225‚Üí3. **`z` is NOT depth** ‚Äî `transform.z` is a generic coordinate. Use `depth` (Depth component) for render order.
   226‚Üí   226‚Üí4. **`Mask` component is Canvas-only in v4.** For WebGL masking, use `enableFilters()` then `filters.internal.addMask()`. `setMask()` logs a warning in WebGL.
   227‚Üí   227‚Üí5. **Flip does NOT affect physics bodies** ‚Äî it's rendering-only. Track facing direction separately in collision code.
   228‚Üí   228‚Üí
   229‚Üí   229‚Üí### Right way (v4 tint fill + WebGL mask)
   230‚Üí   230‚Üí```js
   231‚Üí   231‚Üí// v3: sprite.setTintFill(0xff0000);  // REMOVED in v4
   232‚Üí   232‚Üísprite.setTint(0xff0000).setTintMode(Phaser.TintModes.FILL);  // v4 correct
   233‚Üí   233‚Üí
   234‚Üí   234‚Üí// v4 WebGL mask (BitmapMask is removed)
   235‚Üí   235‚Üísprite.enableFilters();
   236‚Üí   236‚Üísprite.filters.internal.addMask(maskShape);
   237‚Üí   237‚Üí```
   238‚Üí   238‚Üí
   239‚Üí   239‚Üí### Pitfalls
   240‚Üí   240‚Üí- **DO NOT** use `setSize()` thinking it changes visual size ‚Äî it changes internal/native dimensions. Use `setDisplaySize()` for visual size (adjusts scale).
   241‚Üí   241‚Üí- **DO NOT** mix `depth` and `setToTop()`/`setAbove()` ‚Äî `depth` wins after the next sort.
   242‚Üí   242‚Üí- **DO NOT** apply physics bodies to objects with `scrollFactor != 1` ‚Äî physics uses world position, scroll factor offsets rendering only.
   243‚Üí   243‚Üí- **DO NOT** call `enableFilters()` lazily inside `update()` ‚Äî call once in `create()`.

   245‚Üí   245‚Üí- Tint modes in v4: `MULTIPLY` (default), `FILL`, `ADD`, `SCREEN`, `OVERLAY`, `HARD_LIGHT`.
   246‚Üí   246‚Üí

   248‚Üí   248‚Üí- `setTintFill()` removed ‚Üí `setTint(color).setTintMode(FILL)`.
   249‚Üí   249‚Üí- Tint color and tint mode are now separate (`tintMode` property + `setTintMode()` method).

   255‚Üí   255‚Üí## 8. v3-to-v4-migration ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   256‚Üí   256‚Üí
   257‚Üí   257‚Üí### Critical rules (breaking changes)
   258‚Üí   258‚Üí1. **FX (`preFX`/`postFX`) ‚Üí Filters.** `sprite.preFX.addGlow(...)` ‚Üí `sprite.enableFilters(); sprite.filters.internal.addGlow(...)`.
   259‚Üí   259‚Üí2. **`BitmapMask` removed.** Use `filters.internal.addMask(maskObject)`. `GeometryMask` still exists (Canvas only).
   260‚Üí   260‚Üí3. **`setTintFill()` removed.** Use `setTint(color).setTintMode(Phaser.TintModes.FILL)`.
   261‚Üí   261‚Üí4. **`Geom.Point` removed.** Use `Phaser.Math.Vector2`. All geometry classes now return `Vector2` (Circle, Ellipse, Line, Polygon, Rectangle, Triangle).
   262‚Üí   262‚Üí5. **`Math.TAU` value changed.** v3: `PI/2` (incorrect). v4: `PI*2` (correct). `Math.PI2` removed ‚Üí use `Math.TAU`. New: `Math.PI_OVER_2` for `PI/2`.
   263‚Üí   263‚Üí6. **`Phaser.Struct.Set` and `Phaser.Struct.Map` removed** ‚Üí use native `Set` and `Map`. Methods like `iterateLocal`, `contains`, `setAll` are gone.
   264‚Üí   264‚Üí7. **`DynamicTexture`/`RenderTexture` now require explicit `.render()`** ‚Äî drawing commands are buffered, not executed immediately.
   265‚Üí   265‚Üí8. **Pipelines ‚Üí RenderNodes.** Custom WebGL pipelines must be rewritten as render nodes. Do NOT make direct `gl` calls ‚Äî use an `Extern` game object if you need raw WebGL.
   266‚Üí   266‚Üí9. **`Mesh` and `Plane` game objects REMOVED.**
   267‚Üí   267‚Üí10. **`roundPixels` now defaults to `false`** (was `true` in v3). Per-object control via `gameObject.vertexRoundMode`.
   268‚Üí   268‚Üí
   269‚Üí   269‚Üí### Right way (FX ‚Üí Filters, BitmapMask ‚Üí Mask filter)

   281‚Üí   281‚Üíconst mask = new Phaser.Display.Masks.BitmapMask(scene, maskImage);
   282‚Üí   282‚Üísprite.setMask(mask);
   283‚Üí   283‚Üí// v4:
   284‚Üí   284‚Üísprite.enableFilters();

   324‚Üí   324‚Üí## 9. v4-new-features ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   325‚Üí   325‚Üí
   326‚Üí   326‚Üí### Critical rules
   327‚Üí   327‚Üí1. **Filters are WebGL only.** `enableFilters()` returns early on Canvas. Game objects REQUIRE `enableFilters()` first; cameras have filters by default.
   328‚Üí   328‚Üí2. **Internal filters run BEFORE camera transform (object-local); External AFTER (screen space).** Internal is cheaper (object region only); External is full-screen.
   329‚Üí   329‚Üí3. **Filter order matters** ‚Äî each filter receives the output of the previous one.
   330‚Üí   330‚Üí4. **`CaptureFrame` requires `camera.setForceComposite(true)`** ‚Äî otherwise nothing is captured.
   331‚Üí   331‚Üí5. **`SpriteGPULayer` requires single texture + single image per layer.** Populate once; updating buffer is expensive. "Remove" members by setting `scaleX/scaleY/alpha = 0`.
   332‚Üí   332‚Üí
   333‚Üí   333‚Üí### Right way (filters on sprite + camera)
   334‚Üí   334‚Üí```js
   335‚Üí   335‚Üíconst sprite = this.add.sprite(400, 300, 'hero');
   336‚Üí   336‚Üísprite.enableFilters();  // MUST call before accessing .filters
   337‚Üí   337‚Üísprite.filters.internal.addGlow(0x00ff00, 4);  // object-local space
   338‚Üí   338‚Üí
   339‚Üí   339‚Üí// Cameras have filters by default ‚Äî no enableFilters() needed
   340‚Üí   340‚Üíthis.cameras.main.filters.external.addBlur(1, 2, 2, 1);  // screen space
   341‚Üí   341‚Üí```
   342‚Üí   342‚Üí
   343‚Üí   343‚Üí### Pitfalls
   344‚Üí   344‚Üí- **DO NOT** try to change Glow `quality` or `distance` after creation ‚Äî they're immutable. Destroy and recreate.
   345‚Üí   345‚Üí- **DO NOT** expect a dedicated `Bloom` filter ‚Äî use `ParallelFilters` (Threshold + Blur + ADD blend) or `Phaser.Actions.AddEffectBloom()`.
   346‚Üí   346‚Üí- **DO NOT** update `SpriteGPULayer` members every frame ‚Äî buffer rebuilds are expensive. Populate once, mutate `alpha`/`scale` to hide.
   347‚Üí   347‚Üí- **DO NOT** use `TilemapGPULayer` for isometric/hex/staggered maps ‚Äî orthographic only.
   348‚Üí   348‚Üí- **DO NOT** forget `generateLayerDataTexture()` after editing tiles on `TilemapGPULayer` ‚Äî changes won't appear otherwise.
   349‚Üí   349‚Üí- Lighting changes the shader ‚Äî breaks render batches. Group lit objects together.
   350‚Üí   350‚Üí
   351‚Üí   351‚Üí### New v4-only APIs
   352‚Üí   352‚Üí- New GameObjects: `CaptureFrame`, `Gradient`, `Noise`, `NoiseCell2D/3D/4D`, `NoiseSimplex2D/3D`, `SpriteGPULayer`, `Stamp`, `TilemapGPULayer`.
   353‚Üí   353‚Üí- New components: `Lighting`, `RenderSteps`, `RenderNodes`.
   354‚Üí   354‚Üí- New tint modes: `MULTIPLY`, `FILL`, `ADD`, `SCREEN`, `OVERLAY`, `HARD_LIGHT`.
   355‚Üí   355‚Üí- New filters: `Blend`, `Blocky`, `CombineColorMatrix`, `GradientMap`, `ImageLight`, `Key`, `Mask`, `NormalTools`, `PanoramaBlur`, `ParallelFilters`, `Quantize`, `Sampler`, `Threshold`.
   356‚Üí   356‚Üí- `setRenderNodeRole(role, nodeName, data?)` to override render nodes per role (`'Submitter'`, `'Transformer'`, `'Texturer'`).

   360‚Üí   360‚Üí## 10. game-setup-and-config ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   361‚Üí   361‚Üí
   362‚Üí   362‚Üí### Critical rules
   363‚Üí   363‚Üí1. **Default dimensions changed in v4: width=1024, height=768** (was 800x600 in v3).
   364‚Üí   364‚Üí2. **`scale` sub-object takes priority over top-level** `width`/`height`/`zoom`/`parent`. Don't set both ‚Äî leads to confusion.
   365‚Üí   365‚Üí3. **`Phaser.WEBGL` has NO fallback.** Unlike `AUTO`, if WebGL is unsupported the game fails silently. Use `Phaser.AUTO` for safety.
   366‚Üí   366‚Üí4. **`parent: null` vs `parent: undefined`.** `undefined` (or omitted) appends to `document.body`. `null` = no parent ‚Äî you must add the canvas to the DOM yourself.
   367‚Üí   367‚Üí5. **`fps.target` is advisory only.** Use `fps.limit` to actually cap the frame rate. `fps.limit` can only SLOW DOWN, never speed up beyond the display refresh.
   368‚Üí   368‚Üí

   370‚Üí   370‚Üí```js
   371‚Üí   371‚Üíconst config = {

   376‚Üí   376‚Üí    autoCenter: Phaser.Scale.CENTER_BOTH,
   377‚Üí   377‚Üí    width: 1280,
   378‚Üí   378‚Üí    height: 720,

   380‚Üí   380‚Üí  physics: {
   381‚Üí   381‚Üí    default: 'matter',
   382‚Üí   382‚Üí    matter: {
   383‚Üí   383‚Üí      gravity: { y: 1 },
   384‚Üí   384‚Üí      enableSleeping: true,
   385‚Üí   385‚Üí      debug: false,
   386‚Üí   386‚Üí      setBounds: true,           // walls around canvas edges
   387‚Üí   387‚Üí    },
   388‚Üí   388‚Üí  },
   389‚Üí   389‚Üí  input: { gamepad: true },      // enable gamepad

   414‚Üí   414‚Üí## 11. tweens ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   415‚Üí   415‚Üí
   416‚Üí   416‚Üí### Critical rules
   417‚Üí   417‚Üí1. **Tweens auto-destroy after completion** unless `persist: true`. Storing a reference and using it after completion = stale.
   418‚Üí   418‚Üí2. **`repeat` is per-property; `loop` restarts the entire tween.** `repeat: 1` = property plays twice total. `loop: -1` means `onComplete` NEVER fires.
   419‚Üí   419‚Üí3. **`onComplete` never fires for infinite tweens** ‚Äî use `tween.completeAfterLoop(0)` to end gracefully.
   420‚Üí   420‚Üí4. **Seeking suppresses events by default.** `tween.seek(ms)` does not dispatch events unless you pass `true` as the third arg.
   421‚Üí   421‚Üí5. **Tweens skip properties starting with `_`** (underscore prefix). Use this to protect internal fields.
   422‚Üí   422‚Üí
   423‚Üí   423‚Üí### Right way (chain + relative values + callbacks)
   424‚Üí   424‚Üí```js
   425‚Üí   425‚Üíthis.tweens.chain({
   426‚Üí   426‚Üí  targets: this.player,
   427‚Üí   427‚Üí  tweens: [
   428‚Üí   428‚Üí    { x: 300, duration: 1000, ease: 'Power2' },
   429‚Üí   429‚Üí    { y: '+=100', duration: 800, ease: 'Bounce.easeOut' },  // relative
   430‚Üí   430‚Üí    { alpha: 0, duration: 400, onComplete: () => this.player.destroy() },
   431‚Üí   431‚Üí  ],
   432‚Üí   432‚Üí});
   433‚Üí   433‚Üí
   434‚Üí   434‚Üí// Stagger across multiple targets
   435‚Üí   435‚Üíthis.tweens.add({
   436‚Üí   436‚Üí  targets: [s1, s2, s3],
   437‚Üí   437‚Üí  alpha: 0,
   438‚Üí   438‚Üí  duration: 500,
   439‚Üí   439‚Üí  delay: this.tweens.stagger(100, { from: 'center' }),
   440‚Üí   440‚Üí});
   441‚Üí   441‚Üí```
   442‚Üí   442‚Üí
   443‚Üí   443‚Üí### Pitfalls
   444‚Üí   444‚Üí- **DO NOT** use `this.tweens.stagger()` with a single target ‚Äî no visible effect.
   445‚Üí   445‚Üí- **DO NOT** forget to destroy a `persist: true` tween when done ‚Äî it stays in memory.
   446‚Üí   446‚Üí- **DO NOT** set `duration: 0` ‚Äî internally clamped to 0.01ms minimum.
   447‚Üí   447‚Üí- **DO NOT** rely on `onComplete` for `loop: -1` tweens ‚Äî never fires.
   448‚Üí   448‚Üí- **DO NOT** destroy a tween's target without stopping the tween first ‚Äî tween auto-completes early if `target.isDestroyed === true`.
   449‚Üí   449‚Üí- `TweenManager.timeScale` multiplies with `Tween.timeScale` ‚Äî setting either to 0 freezes.
   450‚Üí   450‚Üí- Callbacks receive `(tween, targets)` ‚Äî not `(target)`.
   451‚Üí   451‚Üí- Easing string names are case-insensitive; `'Power2'` = `'Cubic.easeOut'`. Bare names default to `.Out` variant.
   452‚Üí   452‚Üí
   453‚Üí   453‚Üí### v3‚Üív4
   454‚Üí   454‚Üí- No major changes to the Tween API.
   455‚Üí   455‚Üí

   458‚Üí   458‚Üí## 12. time-and-timers ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   459‚Üí   459‚Üí
   460‚Üí   460‚Üí### Critical rules
   461‚Üí   461‚Üí1. **`repeat: N` means N+1 total fires** (1 initial + N repeats). Off-by-one source.
   462‚Üí   462‚Üí2. **`delay: 0` with `repeat`/`loop` throws** `'TimerEvent infinite loop created via zero delay'`. Always use a non-zero delay.
   463‚Üí   463‚Üí3. **`this.time.delayedCall(delay, callback)` is the safe one-shot pattern.** For repeating, use `addEvent({delay, callback, repeat})`.
   464‚Üí   464‚Üí4. **Timelines start PAUSED.** You MUST call `timeline.play()` after `this.add.timeline([...])`.
   465‚Üí   465‚Üí5. **`timer.reset(config)` does NOT re-add to the Clock.** You must call `this.time.addEvent(timer)` again.
   466‚Üí   466‚Üí
   467‚Üí   467‚Üí### Right way (delayed call + repeating timer + remove)
   468‚Üí   468‚Üí```js
   469‚Üí   469‚Üí// One-shot
   470‚Üí   470‚Üíthis.time.delayedCall(1000, () => this.scene.start('GameOver'));
   471‚Üí   471‚Üí
   472‚Üí   472‚Üí// Repeating (5 total fires: 1 initial + 4 repeats)
   473‚Üí   473‚Üíconst timer = this.time.addEvent({
   474‚Üí   474‚Üí  delay: 500,
   475‚Üí   475‚Üí  callback: this.fireBullet,
   476‚Üí   476‚Üí  callbackScope: this,
   477‚Üí   477‚Üí  repeat: 4,
   478‚Üí   478‚Üí});
   479‚Üí   479‚Üí
   480‚Üí   480‚Üí// Stop
   481‚Üí   481‚Üítimer.remove();          // silently
   482‚Üí   482‚Üítimer.remove(true);      // fires callback one last time
   483‚Üí   483‚Üí
   484‚Üí   484‚Üí// Timeline (always starts paused)
   485‚Üí   485‚Üíconst tl = this.add.timeline([
   486‚Üí   486‚Üí  { at: 0, run: () => this.title.setAlpha(1) },
   487‚Üí   487‚Üí  { at: 1000, tween: { targets: this.title, y: 100, duration: 500 } },
   488‚Üí   488‚Üí]);
   489‚Üí   489‚Üítl.play();
   490‚Üí   490‚Üí```
   491‚Üí   491‚Üí
   492‚Üí   492‚Üí### Pitfalls
   493‚Üí   493‚Üí- **DO NOT** omit `callbackScope` with a regular function ‚Äî `this` defaults to the TimerEvent, not the scene. Use arrow functions or `callbackScope: this`.
   494‚Üí   494‚Üí- **DO NOT** expect `timeline.timeScale` to affect its spawned tweens ‚Äî only the timeline's own elapsed counter. Set tween `timeScale` separately.
   495‚Üí   495‚Üí- **DO NOT** use `once: true` on a Timeline event and expect it to reappear on `reset()` or loop ‚Äî it's spliced permanently.
   496‚Üí   496‚Üí- **DO NOT** assume `addEvent()` is immediate ‚Äî pushed to a pending list, active next frame.
   497‚Üí   497‚Üí- **DO NOT** confuse `paused = true` with `timeScale = 0` ‚Äî both freeze timers but `paused` skips the update loop entirely (prefer for full freeze).
   498‚Üí   498‚Üí- Scene pause also pauses the Timeline.
   499‚Üí   499‚Üí
   500‚Üí   500‚Üí### v3‚Üív4
   501‚Üí   501‚Üí- No major changes.

   505‚Üí   505‚Üí## 13. events-system ‚ö° (HIGH-PRIORITY AUDIT TARGET)
   506‚Üí   506‚Üí
   507‚Üí   507‚Üí### Critical rules
   508‚Üí   508‚Üí1. **`on()` does NOT auto-remove.** Every `scene.restart()` adds duplicate listeners ‚Äî memory leak. Always clean up in `shutdown`:
   509‚Üí   509‚Üí   ```js
   510‚Üí   510‚Üí   this.events.once('shutdown', () => this.input.off('pointerdown', this.shoot, this));
   511‚Üí   511‚Üí   ```
   512‚Üí   512‚Üí2. **`off()` requires the EXACT same function reference AND context.** Anonymous arrow functions cannot be removed ‚Äî store a reference or use class methods.
   513‚Üí   513‚Üí3. **`this.game.events` and `this.events` are DIFFERENT emitters.** Game events persist across scene restarts; scene events don't. Clean up game.events listeners on SHUTDOWN.
   514‚Üí   514‚Üí4. **`changedata-{key}` callback args differ from generic `changedata`.** Key-specific: `(parent, value, previousValue)`. Generic: `(parent, key, value, previousValue)`. Easy to miss.
   515‚Üí   515‚Üí5. **Sleeping scenes can still receive events** from other scenes (global registry). Be careful with active listeners on sleeping scenes.
   516‚Üí   516‚Üí
   517‚Üí   517‚Üí### Right way (cleanup on shutdown + cross-scene communication)
   518‚Üí   518‚Üí```js
   519‚Üí   519‚Üícreate() {
   520‚Üí   520‚Üí  this.events.on('update', this.onUpdate, this);     // named method, this context
   521‚Üí   521‚Üí  this.game.events.on('blur', this.onBlur, this);    // game-level persists!
   522‚Üí   522‚Üí
   523‚Üí   523‚Üí  this.events.once('shutdown', () => {
   524‚Üí   524‚Üí    this.events.off('update', this.onUpdate, this);
   525‚Üí   525‚Üí    this.game.events.off('blur', this.onBlur, this);  // CRITICAL
   526‚Üí   526‚Üí  });
   527‚Üí   527‚Üí}
   528‚Üí   528‚Üí
   529‚Üí   529‚Üí// Cross-scene via global registry (NOT scene events)
   530‚Üí   530‚Üíthis.registry.set('score', 100);                          // Scene A
   531‚Üí   531‚Üí// Scene B:
   532‚Üí   532‚Üíthis.registry.events.on('changedata-score', (game, val) => this.scoreText.setText(val));
   533‚Üí   533‚Üí// Cleanup registry listeners too:
   534‚Üí   534‚Üíthis.events.once('shutdown', () => this.registry.events.off('changedata-score', handler));
   535‚Üí   535‚Üí```
   536‚Üí   536‚Üí
   537‚Üí   537‚Üí### Pitfalls
   538‚Üí   538‚Üí- **DO NOT** use arrow literals for listeners you intend to remove: `this.input.on('pointerdown', () => this.shoot())` ‚Äî cannot be removed.
   539‚Üí   539‚Üí- **DO NOT** use `removeAllListeners()` without an arg unless you want to nuke all events on that emitter.
   540‚Üí   540‚Üí- **DO NOT** confuse `shutdown` (scene stops, can restart) with `destroy` (permanent). Restart fires `SHUTDOWN` ‚Üí `START` ‚Üí `CREATE`, never `DESTROY`.
   541‚Üí   541‚Üí- **DO NOT** forget that input events fire in order: Game Object ‚Üí `gameobjectdown` on `this.input` ‚Üí `pointerdown` on `this.input`. `stopPropagation()` halts further levels.
   542‚Üí   542‚Üí- Named constants preferred: `Phaser.Scenes.Events.SHUTDOWN`, `Phaser.Input.Events.POINTER_DOWN`, etc.
   543‚Üí   543‚Üí- Key-suffix pattern: `'filecomplete-image-logo'`, `'animationcomplete-walk'`, `'changedata-score'`.
   544‚Üí   544‚Üí
   545‚Üí   545‚Üí### v3‚Üív4
   546‚Üí   546‚Üí- No major changes; eventemitter3 still underlying.

   550‚Üí   550‚Üí## 14. animations
   551‚Üí   551‚Üí
   552‚Üí   552‚Üí### Critical rules
   553‚Üí   553‚Üí1. **Animations are GLOBAL by default.** `this.anims.create()` registers across all scenes. Do NOT recreate in every scene ‚Äî it logs a warning and returns the existing one. Guard with `if (!this.anims.exists(key))`.
   554‚Üí   554‚Üí2. **`repeat: -1` NEVER fires `animationcomplete`.** Use `stop()` to end infinite animations ‚Äî fires `animationstop` instead.
   555‚Üí   555‚Üí3. **`frameRate` BEATS `duration`.** If both are set, `frameRate` wins. Set only `duration` (leave `frameRate` null) to control total length.
   556‚Üí   556‚Üí4. **Local animations override global.** Same key on a sprite's local map takes priority.
   557‚Üí   557‚Üí5. **Mix delays only work with `play()`.** `playAfterDelay`/`playAfterRepeat` bypass mixes.
   558‚Üí   558‚Üí
   559‚Üí   559‚Üí### Right way (global anims + chained playback)
   560‚Üí   560‚Üí```js
   561‚Üí   561‚Üí// In BootScene or PreloadScene (once):
   562‚Üí   562‚Üíif (!this.anims.exists('walk')) {
   563‚Üí   563‚Üí  this.anims.create({
   564‚Üí   564‚Üí    key: 'walk',
   565‚Üí   565‚Üí    frames: this.anims.generateFrameNames('player', { prefix: 'walk_', start: 1, end: 8, zeroPad: 2 }),
   566‚Üí   566‚Üí    frameRate: 12,
   567‚Üí   567‚Üí    repeat: -1,
   568‚Üí   568‚Üí  });
   569‚Üí   569‚Üí}
   570‚Üí   570‚Üí
   571‚Üí   571‚Üí// In GameScene:
   572‚Üí   572‚Üísprite.play('walk');
   573‚Üí   573‚Üísprite.chain('idle');  // queued after walk stops
   574‚Üí   574‚Üí```
   575‚Üí   575‚Üí
   576‚Üí   576‚Üí### Pitfalls
   577‚Üí   577‚Üí- **DO NOT** expect `animationcomplete` for `repeat: -1` ‚Äî use `animationstop`.
   578‚Üí   578‚Üí- **DO NOT** set both `frameRate` and `duration` expecting duration to win ‚Äî frameRate wins.
   579‚Üí   579‚Üí- **DO NOT** forget per-frame `duration` is ADDITIVE on top of base msPerFrame.
   580‚Üí   580‚Üí- **DO NOT** call `sprite.play(key)` thinking it queues ‚Äî it STOPS the current anim (fires `animationstop`). Use `play(key, true)` to skip if already playing.
   581‚Üí   581‚Üí- **DO NOT** leave chained anims unintentionally ‚Äî clear with `sprite.anims.chain()` (no args).
   582‚Üí   582‚Üí- Chained anims fire after `animationstop` too ‚Äî clear queue before stopping if unwanted.
   583‚Üí   583‚Üí- `generateFrameNumbers` end=-1 means last frame; `__BASE` is excluded automatically.
   584‚Üí   584‚Üí

   586‚Üí   586‚Üí- No breaking changes.

   590‚Üí   590‚Üí## 15. audio-and-sound
   591‚Üí   591‚Üí
   592‚Üí   592‚Üí### Critical rules
   593‚Üí   593‚Üí1. **Sounds are NOT auto-cleaned on scene shutdown.** Looping sounds continue across scene changes unless explicitly stopped. Stop them in `shutdown`.
   594‚Üí   594‚Üí2. **Browser autoplay policy: audio is locked until user interaction.** Phaser handles this automatically; listen for `UNLOCKED` event if you need to know when ready.
   595‚Üí   595‚Üí3. **Always provide multiple audio formats** ‚Äî `this.load.audio('bgm', ['bgm.ogg', 'bgm.mp3'])`. MP3 broadest; OGG lacks Safari; AAC/M4A good for Safari/iOS.
   596‚Üí   596‚Üí4. **HTML5 Audio needs `instances` for simultaneous playback** ‚Äî `this.load.audio('shot', 'shot.mp3', { instances: 4 })`. Default is 1. WebAudio has no such limit.
   597‚Üí   597‚Üí5. **Spatial audio is WebAudio only** ‚Äî silently ignored on HTML5 Audio fallback.
   598‚Üí   598‚Üí
   599‚Üí   599‚Üí### Right way (BGM + SFX + cleanup)
   600‚Üí   600‚Üí```js
   601‚Üí   601‚Üípreload() {
   602‚Üí   602‚Üí  this.load.audio('bgm', ['bgm.ogg', 'bgm.mp3']);
   603‚Üí   603‚Üí  this.load.audio('jump', ['jump.ogg', 'jump.mp3']);
   604‚Üí   604‚Üí}
   605‚Üí   605‚Üícreate() {
   606‚Üí   606‚Üí  this.bgm = this.sound.add('bgm', { loop: true, volume: 0.4 });
   607‚Üí   607‚Üí  this.bgm.play();
   608‚Üí   608‚Üí  this.events.once('shutdown', () => this.bgm?.stop());   // CRITICAL
   609‚Üí   609‚Üí}
   610‚Üí   610‚Üíjump() { this.sound.play('jump'); }  // fire-and-forget (auto-destroys)
   611‚Üí   611‚Üí```
   612‚Üí   612‚Üí
   613‚Üí   613‚Üí### Pitfalls
   614‚Üí   614‚Üí- **DO NOT** use `this.sound.play(key)` if you need to control the sound later ‚Äî it auto-destroys. Use `this.sound.add(key)` for a retained reference.
   615‚Üí   615‚Üí- **DO NOT** rely on `pan` on iOS/Safari ‚Äî `StereoPannerNode` not supported, events fire but no audible effect.
   616‚Üí   616‚Üí- **DO NOT** forget `pauseOnBlur` (default `true`) pauses all sounds when tab loses focus.
   617‚Üí   617‚Üí- **DO NOT** assume `seek` works on a stopped sound ‚Äî no effect.
   618‚Üí   618‚Üí- SoundManager is shared/global ‚Äî `this.sound` in every scene references the same manager.
   619‚Üí   619‚Üí- iOS 17/18+ can interrupt audio on background ‚Äî Phaser handles via `context.suspend()`/`resume()` on `VISIBLE` game event.
   620‚Üí   620‚Üí

   626‚Üí   626‚Üí## 16. data-manager
   627‚Üí   627‚Üí
   628‚Üí   628‚Üí### Critical rules
   629‚Üí   629‚Üí1. **Mutating a stored object/array does NOT trigger `changedata`.** The reference must change. Re-set with a new reference: `sprite.setData('inv', [...inv, 'potion'])`.
   630‚Üí   630‚Üí2. **`values` proxy requires `set()` first.** Direct assignment `sprite.data.values.newKey = 1` does NOT create the event proxy ‚Äî use `sprite.setData('newKey', 1)` first.
   631‚Üí   631‚Üí3. **Registry listeners PERSIST across scene restarts.** Always remove them on SHUTDOWN ‚Äî they are on `game.registry.events`, not `scene.events`.
   632‚Üí   632‚Üí4. **Frozen DataManagers fail SILENTLY.** `set/remove/inc/toggle/pop/merge` all no-op without error or event. Easy to forget you froze the data.
   633‚Üí   633‚Üí5. **`reset()` emits NO events** ‚Äî clears silently. Iterate and `remove()` individually if you need removal events.
   634‚Üí   634‚Üí
   635‚Üí   635‚Üí### Right way (per-GO data + registry + cleanup)
   636‚Üí   636‚Üí```js
   637‚Üí   637‚Üíplayer.setData('hp', 100);
   638‚Üí   638‚Üíplayer.on('changedata-hp', (go, val, prev) => { if (val <= 0) go.destroy(); });
   639‚Üí   639‚Üí
   640‚Üí   640‚Üí// Cross-scene via registry:
   641‚Üí   641‚Üíthis.registry.set('score', 0);
   642‚Üí   642‚Üí// In another scene:
   643‚Üí   643‚Üíconst handler = (game, val) => this.scoreText.setText(val);
   644‚Üí   644‚Üíthis.registry.events.on('changedata-score', handler);
   645‚Üí   645‚Üíthis.events.once('shutdown', () => this.registry.events.off('changedata-score', handler));
   646‚Üí   646‚Üí```
   647‚Üí   647‚Üí
   648‚Üí   648‚Üí### Pitfalls
   649‚Üí   649‚Üí- **DO NOT** confuse `changedata` (4 args: parent, key, value, prev) with `changedata-{key}` (3 args: parent, value, prev) ‚Äî key is omitted in the latter.
   650‚Üí   650‚Üí- **DO NOT** use `inc()` on non-numeric keys ‚Äî `+` operator concatenates strings, coerces booleans.
   651‚Üí   651‚Üí- **DO NOT** expect scene data to reset on restart ‚Äî `DataManagerPlugin` removes its shutdown listener but does NOT clear data. Manually `this.data.reset()` on shutdown if needed.
   652‚Üí   652‚Üí- **DO NOT** confuse `get()` (returns copy for primitives, reference for objects) with `values` (live proxy that emits events on assignment).
   653‚Üí   653‚Üí- Keys are case-sensitive ‚Äî `'gold'` ‚âÝ `'Gold'`.
   654‚Üí   654‚Üí- `getAll()` returns a shallow copy ‚Äî primitive values are independent, but nested objects share references.

   661‚Üí   661‚Üí## 17. filters-and-postfx
   662‚Üí   662‚Üí
   663‚Üí   663‚Üí### Critical rules
   664‚Üí   664‚Üí1. **Filters are WebGL only.** `enableFilters()` returns early if WebGL is unavailable.
   665‚Üí   665‚Üí2. **Game objects REQUIRE `enableFilters()` before accessing `filters`.** Cameras have filters by default.
   666‚Üí   666‚Üí3. **Internal = object-local space (cheaper, region-sized); External = screen space (full-screen, more expensive).** Use internal wherever possible.
   667‚Üí   667‚Üí4. **Filter order matters** ‚Äî output of one feeds the next.
   668‚Üí   668‚Üí5. **Glow `quality` and `distance` are immutable** ‚Äî destroy and recreate the filter to change them.
   669‚Üí   669‚Üí
   670‚Üí   670‚Üí### Right way (filters on sprite + camera)
   671‚Üí   671‚Üí```js
   672‚Üí   672‚Üíconst sprite = this.add.sprite(400, 300, 'enemy');
   673‚Üí   673‚Üísprite.enableFilters();
   674‚Üí   674‚Üíconst glow = sprite.filters.internal.addGlow(0x00ff00, 4);
   675‚Üí   675‚Üíglow.outerStrength = 8;        // mutable
   676‚Üí   676‚Üíglow.setActive(false);         // toggle without removing
   677‚Üí   677‚Üísprite.filters.internal.remove(glow);
   678‚Üí   678‚Üí
   679‚Üí   679‚Üí// Camera filters (no enableFilters() needed):
   680‚Üí   680‚Üíthis.cameras.main.filters.external.addVignette(0.5, 0.5, 0.5, 0.8);
   681‚Üí   681‚Üí
   682‚Üí   682‚Üí// Bloom via ParallelFilters (no dedicated Bloom filter):
   683‚Üí   683‚Üíconst pf = this.cameras.main.filters.internal.addParallelFilters();
   684‚Üí   684‚Üípf.top.addThreshold(0.5, 1);
   685‚Üí   685‚Üípf.top.addBlur();
   686‚Üí   686‚Üípf.blend.blendMode = Phaser.BlendModes.ADD;
   687‚Üí   687‚Üípf.blend.amount = 0.5;
   688‚Üí   688‚Üí```
   689‚Üí   689‚Üí
   690‚Üí   690‚Üí### Pitfalls
   691‚Üí   691‚Üí- **DO NOT** use `setMask()` in v4 WebGL ‚Äî logs warning. Use `filters.internal.addMask()`.
   692‚Üí   692‚Üí- **DO NOT** forget `camera.setForceComposite(true)` for `CaptureFrame`.
   693‚Üí   693‚Üí- **DO NOT** expect `BitmapMask` ‚Äî removed in v4. Use `FilterMask`.
   694‚Üí   694‚Üí- **DO NOT** leave many objects with active filters ‚Äî each creates extra draw calls. Performance-test early.
   695‚Üí   695‚Üí- **DO NOT** forget to set `mask.autoUpdate = false` + `mask.needsUpdate = true` for static game-object masks (default re-renders each frame).
   696‚Üí   696‚Üí- `ignoreDestroy = true` on a controller lets it survive FilterList destruction for reuse ‚Äî manage lifecycle manually.
   697‚Üí   697‚Üí- v3 derived FX (`Bloom`, `Shine`, `Circle`, `Gradient`) removed ‚Äî use Actions or new GameObjects.
   698‚Üí   698‚Üí
   699‚Üí   699‚Üí### v3‚Üív4
   700‚Üí   700‚Üí- `preFX`/`postFX` ‚Üí `filters.internal`/`filters.external`.
   701‚Üí   701‚Üí- `BitmapMask` removed ‚Üí `FilterMask`.
   702‚Üí   702‚Üí- Glow `quality` semantics changed: now an integer (default 10), not 0-1 fraction.

   707‚Üí   707‚Üí## 18. geometry-and-math
   708‚Üí   708‚Üí
   709‚Üí   709‚Üí### Critical rules
   710‚Üí   710‚Üí1. **`Geom.Point` is REMOVED in v4.** Use `Phaser.Math.Vector2` or plain `{x, y}`. The `GEOM_CONST.POINT` constant still exists but the class doesn't.
   711‚Üí   711‚Üí2. **Angles are in RADIANS throughout the math API.** Use `Phaser.Math.DegToRad()` / `RadToDeg()` for conversion. The `Phaser.Math.TAU` constant (2*PI) is new in v4.
   712‚Üí   712‚Üí3. **Vector2 methods MUTATE in place** and return `this`. Call `.clone()` first to preserve the original.
   713‚Üí   713‚Üí4. **EaseMap short names default to `.Out` variant.** `'Quad'` = `Quad.Out`, not `Quad.In`. Use full `'Quad.easeIn'`.
   714‚Üí   714‚Üí5. **Squared distance is faster** than Euclidean (avoids `Math.sqrt`). Use `Distance.Squared` or `vec.distanceSq()` for comparisons.
   715‚Üí   715‚Üí
   716‚Üí   716‚Üí### Right way (vector math + intersection)
   717‚Üí   717‚Üí```js
   718‚Üí   718‚Üíconst v = new Phaser.Math.Vector2(3, 4);
   719‚Üí   719‚Üív.length();      // 5
   720‚Üí   720‚Üíconst result = v.clone().normalize().scale(10);  // preserve v
   721‚Üí   721‚Üí
   722‚Üí   722‚Üíconst rect = new Phaser.Geom.Rectangle(10, 20, 200, 100);
   723‚Üí   723‚Üírect.contains(50, 50);    // true
   724‚Üí   724‚ÜíPhaser.Geom.Intersects.RectangleToRectangle(rectA, rectB);
   725‚Üí   725‚Üí
   726‚Üí   726‚Üíconst angle = Phaser.Math.Angle.Between(x1, y1, x2, y2);  // radians
   727‚Üí   727‚Üíconst clamped = Phaser.Math.Clamp(value, 0, 100);
   728‚Üí   728‚Üí```
   729‚Üí   729‚Üí
   730‚Üí   730‚Üí### Pitfalls
   731‚Üí   731‚Üí- **DO NOT** confuse `Phaser.Math.TAU` (v4 = 2*PI) with v3 (was PI/2 incorrectly). If migrating v3 code expecting PI/2, use `Phaser.Math.PI_OVER_2`.
   732‚Üí   732‚Üí- **DO NOT** use `Math.PI2` ‚Äî removed. Use `Math.TAU`.
   733‚Üí   733‚Üí- **DO NOT** allocate Vector2 in hot loops ‚Äî reuse output objects via `out` parameters.
   734‚Üí   734‚Üí- **DO NOT** use Geom objects as game objects ‚Äî they're pure data. Render via Graphics or Shape game objects.

   738‚Üí   738‚Üí### v3‚Üív4

   746‚Üí   746‚Üí## 19. graphics-and-shapes
   747‚Üí   747‚Üí
   748‚Üí   748‚Üí### Critical rules
   749‚Üí   749‚Üí1. **Graphics `arc()` uses RADIANS; `this.add.arc()` Shape factory uses DEGREES.** Mixing these up is the most common bug.
   750‚Üí   750‚Üí2. **Set style BEFORE drawing.** `fillStyle()` / `lineStyle()` must precede the corresponding `fill*` / `stroke*` call. They are NOT retroactive.
   751‚Üí   751‚Üí3. **Graphics has NO Origin or GetBounds component** ‚Äî use `setPosition(x, y)` and `displayOriginX`/`displayOriginY`. Shape game objects DO have Origin and GetBounds.
   752‚Üí   752‚Üí4. **`generateTexture()` uses the Canvas API.** Gradient fills (`fillGradientStyle`) will NOT appear in generated textures.
   753‚Üí   753‚Üí5. **Shape objects do NOT support `setTint()`** ‚Äî use `setFillStyle(color, alpha)` / `setStrokeStyle(lineWidth, color, alpha)`.
   754‚Üí   754‚Üí
   755‚Üí   755‚Üí### Right way (imperative drawing vs Shape object)
   756‚Üí   756‚Üí```js
   757‚Üí   757‚Üí// Graphics (dynamic, multiple shapes, paths, gradients):
   758‚Üí   758‚Üíconst gfx = this.add.graphics();
   759‚Üí   759‚Üígfx.fillStyle(0xff0000, 1);
   760‚Üí   760‚Üígfx.fillRect(50, 50, 200, 100);
   761‚Üí   761‚Üígfx.lineStyle(3, 0x00ff00, 1);
   762‚Üí   762‚Üígfx.strokeCircle(400, 150, 60);
   763‚Üí   763‚Üí
   764‚Üí   764‚Üí// Shape object (single shape, full GO features, can be tweened/physics-enabled):
   765‚Üí   765‚Üíconst rect = this.add.rectangle(150, 100, 200, 100, 0x00aa00);
   766‚Üí   766‚Üírect.setStrokeStyle(3, 0xff0000);  // isStroked defaults to false ‚Äî must call explicitly
   767‚Üí   767‚Üí```
   768‚Üí   768‚Üí
   769‚Üí   769‚Üí### Pitfalls
   770‚Üí   770‚Üí- **DO NOT** expect `Shape` objects to support tint methods ‚Äî they don't.
   771‚Üí   771‚Üí- **DO NOT** expect `Graphics` to have origin ‚Äî it doesn't. Position via `setPosition`.
   772‚Üí   772‚Üí- **DO NOT** forget `setStrokeStyle()` on Shapes ‚Äî `isStroked` defaults to `false`.
   773‚Üí   773‚Üí- **DO NOT** use `Graphics` for static shapes ‚Äî `generateTexture()` + Sprite is faster. Graphics replays the command buffer and rebuilds geometry each frame.
   774‚Üí   774‚Üí- **DO NOT** use `IsoBox`/`IsoTriangle` with strokes ‚Äî fill-only.
   775‚Üí   775‚Üí- **DO NOT** use `Line` shape with fill ‚Äî stroke-only (constructor takes `strokeColor` not `fillColor`).
   776‚Üí   776‚Üí- v4: Grid uses `stroke` properties (renamed from `outline` in v3). Rectangle supports rounded corners.
   777‚Üí   777‚Üí- `pathDetailThreshold` (v4 new) skips nearby vertices in Graphics WebGL paths.
   778‚Üí   778‚Üí
   779‚Üí   779‚Üí### v3‚Üív4
   780‚Üí   780‚Üí- Grid shape: `outline` properties renamed to `stroke`.
   781‚Üí   781‚Üí- Rectangle now supports rounded corners.
   782‚Üí   782‚Üí- New `pathDetailThreshold` config for Graphics.
   783‚Üí   783‚Üí

   786‚Üí   786‚Üí## 20. loading-assets
   787‚Üí   787‚Üí
   788‚Üí   788‚Üí### Critical rules
   789‚Üí   789‚Üí1. **Keys are unique per type.** A second `this.load.image('bg', ...)` is silently ignored if `'bg'` already exists. Remove the old texture first to replace.
   790‚Üí   790‚Üí2. **Forgetting `this.load.start()` outside `preload`.** If you call load methods in `create()` or later, the Loader does NOT auto-start ‚Äî call `this.load.start()` manually.
   791‚Üí   791‚Üí3. **`setPath` appends a trailing slash; setting `this.load.path` directly requires you to include it.**
   792‚Üí   792‚Üí4. **File keys include the prefix.** `this.load.setPrefix('MENU.')` + `this.load.image('bg', ...)` ‚Üí cache key is `'MENU.bg'`. Use the full key when referencing.
   793‚Üí   793‚Üí5. **Sprite sheet ‚âÝ atlas.** Use `spritesheet()` for fixed-size grids (referenced by index). Use `atlas()` for packed atlases with named frames.
   794‚Üí   794‚Üí
   795‚Üí   795‚Üí### Right way (preload + outside-preload + chain-loading)
   796‚Üí   796‚Üí```js
   797‚Üí   797‚Üípreload() {
   798‚Üí   798‚Üí  this.load.image('sky', 'assets/sky.png');
   799‚Üí   799‚Üí  this.load.spritesheet('dude', 'assets/dude.png', { frameWidth: 32, frameHeight: 48 });
   800‚Üí   800‚Üí  this.load.audio('jump', ['assets/jump.ogg', 'assets/jump.mp3']);
   801‚Üí   801‚Üí}
   802‚Üí   802‚Üí
   803‚Üí   803‚Üí// Outside preload:
   804‚Üí   804‚Üícreate() {
   805‚Üí   805‚Üí  this.load.image('extra', 'assets/extra.png');
   806‚Üí   806‚Üí  this.load.once('complete', () => this.add.image(400, 300, 'extra'));
   807‚Üí   807‚Üí  this.load.start();  // CRITICAL
   808‚Üí   808‚Üí}
   809‚Üí   809‚Üí
   810‚Üí   810‚Üí// Chained loads:
   811‚Üí   811‚Üípreload() {
   812‚Üí   812‚Üí  this.load.json('level1', 'level1.json');
   813‚Üí   813‚Üí  this.load.on('filecomplete-json-level1', (key, type, data) => {
   814‚Üí   814‚Üí    this.load.image(data.images);  // batch-load from JSON
   815‚Üí   815‚Üí  });
   816‚Üí   816‚Üí}
   817‚Üí   817‚Üí```
   818‚Üí   818‚Üí
   819‚Üí   819‚Üí### Pitfalls
   820‚Üí   820‚Üí- **DO NOT** expect `update()` to fire during `preload()` ‚Äî it's paused. `preupdate`, `postupdate`, and `render` still fire.
   821‚Üí   821‚Üí- **DO NOT** expect progress to monotonically increase ‚Äî adding files mid-load can decrease it.
   822‚Üí   822‚Üí- **DO NOT** forget `crossOrigin: 'anonymous'` when loading from another domain (especially for WebGL textures).
   823‚Üí   823‚Üí- **DO NOT** confuse `imageLoadType: 'XHR'` (default, blob) with `'HTMLImageElement'` ‚Äî the latter helps with CORS in some environments.
   824‚Üí   824‚Üí- **DO NOT** expect "Collection of Images" tilesets to work ‚Äî Tiled parser requires single image per tileset.
   825‚Üí   825‚Üí- The `maxRetries` property (default 2) is per-file at creation; adjusting after has no effect on existing files.
   826‚Üí   826‚Üí- Scene `pack` config loads files BEFORE `preload()` ‚Äî good for progress bar assets.
   827‚Üí   827‚Üí
   828‚Üí   828‚Üí### v3‚Üív4
   829‚Üí   829‚Üí- Default dimensions changed (1024x768 from 800x600).
   830‚Üí   830‚Üí- `loader.maxRetries` default 2 (was 0 in early v3).
   831‚Üí   831‚Üí

   834‚Üí   834‚Üí## 21. particles
   835‚Üí   835‚Üí
   836‚Üí   836‚Üí### Critical rules
   837‚Üí   837‚Üí1. **No `ParticleEmitterManager`** ‚Äî removed since v3.60. `this.add.particles()` returns a `ParticleEmitter` directly.
   838‚Üí   838‚Üí2. **`speed` vs `speedX`/`speedY`:** setting `speed` deactivates `speedY` (radial mode). Setting `speedX`/`speedY` switches to point mode (`radial: false`).
   839‚Üí   839‚Üí3. **`color` overrides `tint`** ‚Äî mutually exclusive; `color` (array) takes priority.
   840‚Üí   840‚Üí4. **`frequency: 0` means emit every frame** (max rate), NOT "never." Use `emitting: false` to prevent emission.
   841‚Üí   841‚Üí5. **`frequency: -1` = explode mode** ‚Äî won't flow automatically. Use `explode()` for bursts.
   842‚Üí   842‚Üí
   843‚Üí   843‚Üí### Right way (flow + burst + zones)
   844‚Üí   844‚Üí```js
   845‚Üí   845‚Üí// Flow emitter
   846‚Üí   846‚Üíconst emitter = this.add.particles(400, 300, 'spark', {
   847‚Üí   847‚Üí  speed: 100, lifespan: 2000,
   848‚Üí   848‚Üí  scale: { start: 1, end: 0, ease: 'cubic.out' },
   849‚Üí   849‚Üí  alpha: { start: 1, end: 0 },
   850‚Üí   850‚Üí  emitZone: { type: 'random', source: new Phaser.Geom.Circle(0, 0, 50) },
   851‚Üí   851‚Üí});
   852‚Üí   852‚Üí
   853‚Üí   853‚Üí// Burst
   854‚Üí   854‚Üíconst burst = this.add.particles(400, 300, 'spark', {
   855‚Üí   855‚Üí  speed: { min: 100, max: 300 }, lifespan: 1000, emitting: false,
   856‚Üí   856‚Üí});
   857‚Üí   857‚Üíburst.explode(30);
   858‚Üí   858‚Üí
   859‚Üí   859‚Üí// Death zone (kill particles leaving a circle)
   860‚Üí   860‚Üíemitter.addDeathZone({ type: 'onLeave', source: new Phaser.Geom.Circle(400, 300, 200) });
   861‚Üí   861‚Üí
   862‚Üí   862‚Üí// Follow target
   863‚Üí   863‚Üíemitter.startFollow(player, 0, -20, true);  // x, y offset, trackVisible
   864‚Üí   864‚Üí```
   865‚Üí   865‚Üí
   866‚Üí   866‚Üí### Pitfalls
   867‚Üí   867‚Üí- **DO NOT** set both `moveToX` and `moveToY` partially ‚Äî both must be set to activate; overrides `angle` and `speed`.
   868‚Üí   868‚Üí- **DO NOT** confuse `emitting: false` (no new particles, alive ones update) with `active: false` (entire emitter frozen).
   869‚Üí   869‚Üí- **DO NOT** confuse `'stop'` event (emission stopped) with `'complete'` (last alive particle died).
   870‚Üí   870‚Üí- **DO NOT** expect `frequency: 0` to mean "never" ‚Äî it means every frame.
   871‚Üí   871‚Üí- **DO NOT** set `maxParticles` thinking it limits alive count ‚Äî it limits TOTAL objects. Use `maxAliveParticles` for visible limit.
   872‚Üí   872‚Üí- `hold` freezes a particle at end of life for N ms before dying.
   873‚Üí   873‚Üí- `advance` pre-warms the emitter so particles are visible on first frame.
   874‚Üí   874‚Üí- `reserve(count)` pre-allocates particles to avoid GC spikes.
   875‚Üí   875‚Üí
   876‚Üí   876‚Üí### v3‚Üív4
   877‚Üí   877‚Üí- No major API changes since 3.60.
   878‚Üí   878‚Üí

   881‚Üí   881‚Üí## 22. physics-arcade
   882‚Üí   882‚Üí
   883‚Üí   883‚Üí### Critical rules
   884‚Üí   884‚Üí1. **Static bodies do NOT auto-sync.** After changing position, scale, or origin of a static body's Game Object, call `body.reset()` or `gameObject.refreshBody()`.
   885‚Üí   885‚Üí2. **World events require opt-in.** `'collide'`, `'overlap'`, `'worldbounds'` only fire if `body.onCollide` / `body.onOverlap` / `body.onWorldBounds` is `true`.
   886‚Üí   886‚Üí3. **`collide` separates bodies; `overlap` only detects.** Use `overlap` for triggers/pickups.
   887‚Üí   887‚Üí4. **Persistent Colliders (`physics.add.collider`) checked every frame; one-shot `physics.collide()` must be called in `update()` each frame.**
   888‚Üí   888‚Üí5. **Collision categories: 32 max.** Default category `0x0001`, mask `1`. PhysicsGroup defaults to mask `2147483647`. After changing categories, call `resetCollisionCategory()` to set mask to all bits.
   889‚Üí   889‚Üí
   890‚Üí   890‚Üí### Right way (static body + categories + collider)
   891‚Üí   891‚Üí```js
   892‚Üí   892‚Üíthis.platforms = this.physics.add.staticGroup();
   893‚Üí   893‚Üíthis.platforms.create(400, 568, 'ground').setScale(2).refreshBody();  // CRITICAL: refreshBody
   894‚Üí   894‚Üí
   895‚Üí   895‚Üíconst CAT_PLAYER = this.physics.nextCategory();
   896‚Üí   896‚Üíconst CAT_ENEMY  = this.physics.nextCategory();
   897‚Üí   897‚Üíplayer.setCollisionCategory(CAT_PLAYER);
   898‚Üí   898‚Üíenemy.setCollisionCategory(CAT_ENEMY);
   899‚Üí   899‚Üíplayer.setCollidesWith([CAT_ENEMY]);
   900‚Üí   900‚Üí
   901‚Üí   901‚Üíthis.physics.add.collider(player, platforms);
   902‚Üí   902‚Üíthis.physics.add.overlap(player, coins, collectCoin, null, this);
   903‚Üí   903‚Üí
   904‚Üí   904‚Üí// World bounds event
   905‚Üí   905‚Üíplayer.body.onWorldBounds = true;
   906‚Üí   906‚Üíthis.physics.world.on('worldbounds', (body, up, down, left, right) => { /* ... */ });
   907‚Üí   907‚Üí```
   908‚Üí   908‚Üí
   909‚Üí   909‚Üí### Pitfalls
   910‚Üí   910‚Üí- **DO NOT** leave `debug: true` in production ‚Äî expensive.
   911‚Üí   911‚Üí- **DO NOT** use `useDamping: true` with large drag values ‚Äî drag becomes a multiplier (e.g., `0.05` = keeps 5%/sec).
   912‚Üí   912‚Üí- **DO NOT** confuse `immovable` (never moved by collisions) with `pushable = false` (reflects velocity to collider but still separates).
   913‚Üí   913‚Üí- **DO NOT** change PhysicsGroup `defaults` expecting existing members to update ‚Äî defaults apply only at member creation.
   914‚Üí   914‚Üí- **DO NOT** use `overlapOnly` with TilemapLayer expecting per-tile collision settings ‚Äî every tile is checked.
   915‚Üí   915‚Üí- **DO NOT** forget `customUpdate: true` requires manual `this.physics.world.update(time, delta)` calls.
   916‚Üí   916‚Üí- RTree (`useTree: true` default) becomes expensive to rebuild with 5000+ dynamic bodies ‚Äî consider `useTree: false`.
   917‚Üí   917‚Üí
   918‚Üí   918‚Üí### v3‚Üív4
   919‚Üí   919‚Üí- No major changes.

   923‚Üí   923‚Üí## 23. render-textures
   924‚Üí   924‚Üí
   925‚Üí   925‚Üí### Critical rules
   926‚Üí   926‚Üí1. **`render()` is REQUIRED.** v4 buffers all drawing commands; nothing appears until `render()` is called (unless `renderMode: 'all'` or `'redraw'`).
   927‚Üí   927‚Üí2. **Command buffer CLEARS after `render()` by default.** Call `preserve(true)` to keep commands for replay.
   928‚Üí   928‚Üí3. **`stamp()` ignores the internal camera** ‚Äî only `draw()` and `capture()` respect the `.camera` property. Texture stamps are positioned absolutely.
   929‚Üí   929‚Üí4. **`resize()` erases content** ‚Äî destroys and recreates the framebuffer.
   930‚Üí   930‚Üí5. **`saveTexture(key)` RENAMES, does not copy.** Calling again with a different key renames. Destroying the RenderTexture without saving first destroys the texture.
   931‚Üí   931‚Üí
   932‚Üí   932‚Üí### Right way (RT + DynamicTexture + minimap)
   933‚Üí   933‚Üí```js
   934‚Üí   934‚Üí// RenderTexture (visible game object)
   935‚Üí   935‚Üíconst rt = this.add.renderTexture(0, 0, 256, 256).setOrigin(0);
   936‚Üí   936‚Üírt.clear();
   937‚Üí   937‚Üírt.fill(0x000000, 0.5);
   938‚Üí   938‚Üírt.draw(sprite, 128, 128);
   939‚Üí   939‚Üírt.render();  // CRITICAL
   940‚Üí   940‚Üí
   941‚Üí   941‚Üí// DynamicTexture (shared texture, cross-scene)
   942‚Üí   942‚Üíconst dt = this.textures.addDynamicTexture('composite', 512, 512);
   943‚Üí   943‚Üídt.stamp('coin', null, 64, 64, { scale: 2, angle: 45 });
   944‚Üí   944‚Üídt.render();
   945‚Üí   945‚Üíthis.add.image(400, 300, 'composite');
   946‚Üí   946‚Üí
   947‚Üí   947‚Üí// Auto-render minimap
   948‚Üí   948‚Üíthis.minimap = this.add.renderTexture(700, 50, 150, 100);
   949‚Üí   949‚Üíthis.minimap.setScrollFactor(0).setRenderMode('all').preserve(true);
   950‚Üí   950‚Üí```
   951‚Üí   951‚Üí
   952‚Üí   952‚Üí### Pitfalls
   953‚Üí   953‚Üí- **DO NOT** draw a DynamicTexture to itself ‚Äî silently skips.
   954‚Üí   954‚Üí- **DO NOT** use `alpha`/`tint` params on `draw()` with game objects ‚Äî they only apply to Texture Frames/strings. Game objects use their own alpha/tint.
   955‚Üí   955‚Üí- **DO NOT** snapshot every frame ‚Äî `readPixels` blocks the GPU pipeline. Use `snapshotPixel` (cheaper) or cache.
   956‚Üí   956‚Üí- **DO NOT** expect anti-aliasing on WebGL1 framebuffers ‚Äî jagged edges. Pre-render sprites instead.
   957‚Üí   957‚Üí- **DO NOT** forget to handle WebGL context loss ‚Äî DynamicTexture contents are lost. Listen for `restorewebgl`.
   958‚Üí   958‚Üí- Default sizes differ: RenderTexture 32x32, DynamicTexture 256x256.
   959‚Üí   959‚Üí- `forceEven` rounds dimensions up by default ‚Äî pass `false` for exact odd dimensions.
   960‚Üí   960‚Üí
   961‚Üí   961‚Üí### v3‚Üív4
   962‚Üí   962‚Üí- **BREAKING:** Drawing commands are now BUFFERED. Must call `.render()` explicitly. In v3, draw commands executed immediately.
   963‚Üí   963‚Üí- New: `preserve()`, `callback()`, `capture()`, `renderMode` ('render'/'redraw'/'all').
   964‚Üí   964‚Üí- `TextureManager.addDynamicTexture` now has `forceEven` parameter.

   968‚Üí   968‚Üí## 24. scale-and-responsive
   969‚Üí   969‚Üí
   970‚Üí   970‚Üí### Critical rules
   971‚Üí   971‚Üí1. **Parent element must have calculable dimensions.** Unstyled `<div>` has zero height ‚Äî breaks centering and scaling. Use `expandParent: true` (default) or set explicit CSS.
   972‚Üí   972‚Üí2. **Do NOT style the canvas directly.** ScaleManager controls `canvas.style.width/height/marginLeft/marginTop`. External CSS conflicts.
   973‚Üí   973‚Üí3. **Fullscreen MUST be triggered from `pointerup` (not `pointerdown`)** ‚Äî touch devices block `pointerdown` requests.
   974‚Üí   974‚Üí4. **`resize()` is for NONE mode only; `setGameSize()` for FIT/ENVELOP/etc.** Mixing them up breaks scaling.
   975‚Üí   975‚Üí5. **RESIZE mode = 1:1 pixel mapping** ‚Äî large displays cause GPU fill-rate pressure on low-end devices.
   976‚Üí   976‚Üí
   977‚Üí   977‚Üí### Right way (FIT + center + fullscreen)
   978‚Üí   978‚Üí```js
   979‚Üí   979‚Üíconst config = {
   980‚Üí   980‚Üí  scale: {
   981‚Üí   981‚Üí    parent: 'game-container',
   982‚Üí   982‚Üí    mode: Phaser.Scale.FIT,
   983‚Üí   983‚Üí    autoCenter: Phaser.Scale.CENTER_BOTH,
   984‚Üí   984‚Üí    width: 1280, height: 720,
   985‚Üí   985‚Üí    min: { width: 640, height: 360 },
   986‚Üí   986‚Üí    max: { width: 1920, height: 1080 },
   987‚Üí   987‚Üí  },
   988‚Üí   988‚Üí  scene: MyScene,
   989‚Üí   989‚Üí};
   990‚Üí   990‚Üí
   991‚Üí   991‚Üí// In scene:
   992‚Üí   992‚Üíthis.scale.on('resize', (gameSize, baseSize, displaySize) => {
   993‚Üí   993‚Üí  this.cameras.resize(gameSize.width, gameSize.height);
   994‚Üí   994‚Üí  // Reposition UI
   995‚Üí   995‚Üí});
   996‚Üí   996‚Üí
   997‚Üí   997‚Üí// Fullscreen (pointerup, not pointerdown):
   998‚Üí   998‚Üíthis.input.on('pointerup', () => this.scale.toggleFullscreen());
   999‚Üí   999‚Üí```
  1000‚Üí  1000‚Üí
  1001‚Üí  1001‚Üí### Pitfalls
  1002‚Üí  1002‚Üí- **DO NOT** apply padding to the parent ‚Äî ScaleManager doesn't account for it. Use a wrapper element.
  1003‚Üí  1003‚Üí- **DO NOT** use `resize()` in FIT mode ‚Äî use `setGameSize()`.
  1004‚Üí  1004‚Üí- **DO NOT** expect fullscreen to work in iframes without `allowfullscreen` attribute.
  1005‚Üí  1005‚Üí- **DO NOT** set `width: '100%'` without a sized parent ‚Äî falls back to `window.innerWidth/innerHeight`.
  1006‚Üí  1006‚Üí- `resizeInterval` (default 500ms) is a fallback poll on top of browser resize events.
  1007‚Üí  1007‚Üí- iOS Safari height quirks handled via `GetInnerHeight` workaround.

  1009‚Üí  1009‚Üí### v3‚Üív4
  1010‚Üí  1010‚Üí- New `Phaser.Scale.EXPAND` mode (value 6) ‚Äî hybrid of RESIZE + FIT.

  1014‚Üí  1014‚Üí## 25. sprites-and-images
  1015‚Üí  1015‚Üí
  1016‚Üí  1016‚Üí### Critical rules
  1017‚Üí  1017‚Üí1. **Use `Image` instead of `Sprite` when you don't need animation.** Sprites are added to the update list and run `preUpdate` every frame. Images skip this overhead.
  1018‚Üí  1018‚Üí2. **`z` does NOT control render order.** Use `depth` (Depth component) or `setToTop()`/`setAbove()` etc.
  1019‚Üí  1019‚Üí3. **`scale` getter returns the AVERAGE of `scaleX` and `scaleY`.** Check `scaleX`/`scaleY` individually when non-uniform.
  1020‚Üí  1020‚Üí4. **`setFrame()` resets size and origin by default.** Pass `false, false` to prevent: `setFrame('f', false, false)`.
  1021‚Üí  1021‚Üí5. **Setting `alpha = 0` or `scale = 0` clears a render flag** ‚Äî the object is NOT drawn at all. Use `setVisible(false)` if you want to explicitly hide without changing alpha.
  1022‚Üí  1022‚Üí
  1023‚Üí  1023‚Üí### Right way (chained setup)
  1024‚Üí  1024‚Üí```js
  1025‚Üí  1025‚Üíconst player = this.add.sprite(100, 200, 'player', 'idle-0')
  1026‚Üí  1026‚Üí  .setScale(2)
  1027‚Üí  1027‚Üí  .setOrigin(0.5, 1)        // bottom-center
  1028‚Üí  1028‚Üí  .setDepth(10)
  1029‚Üí  1029‚Üí  .setFlip(true, false)
  1030‚Üí  1030‚Üí  .setAlpha(0.8)
  1031‚Üí  1031‚Üí  .play('idle');
  1032‚Üí  1032‚Üí
  1033‚Üí  1033‚Üí// Change frame without resetting size/origin:
  1034‚Üí  1034‚Üíplayer.setFrame('walk-1', false, false);
  1035‚Üí  1035‚Üí```
  1036‚Üí  1036‚Üí
  1037‚Üí  1037‚Üí### Pitfalls
  1038‚Üí  1038‚Üí- **DO NOT** use `setTintFill()` ‚Äî REMOVED in v4. Use `setTint(color).setTintMode(Phaser.TintModes.FILL)`.
  1039‚Üí  1039‚Üí- **DO NOT** expect flip to affect physics bodies ‚Äî rendering-only.
  1040‚Üí  1040‚Üí- **DO NOT** set `scrollFactor` on physics-bodied objects expecting collisions to follow ‚Äî physics uses world position; scroll factor offsets rendering only.
  1041‚Üí  1041‚Üí- **DO NOT** use Sprite for static decor ‚Äî Image is cheaper.
  1042‚Üí  1042‚Üí- **DO NOT** assume `rotation` is in degrees ‚Äî it's radians. Use `angle` for degrees.
  1043‚Üí  1043‚Üí- Rotation direction: right-hand clockwise (0=right, 90¬∞=down, 180¬∞=left, -90¬∞=up).
  1044‚Üí  1044‚Üí- `ignoreDestroy = true` prevents destruction ‚Äî you manage lifecycle.
  1045‚Üí  1045‚Üí- `isDestroyed` boolean tracks destroy() state.
  1046‚Üí  1046‚Üí
  1047‚Üí  1047‚Üí### v3‚Üív4
  1048‚Üí  1048‚Üí- `setTintFill()` removed.
  1049‚Üí  1049‚Üí- Tint color and tint mode are now separate properties.
  1050‚Üí  1050‚Üí- TileSprite supports atlas frames and `tileRotation` (new in v4).
  1051‚Üí  1051‚Üí- TileSprite no longer supports texture cropping.

  1055‚Üí  1055‚Üí## 26. tilemaps
  1056‚Üí  1056‚Üí
  1057‚Üí  1057‚Üí### Critical rules
  1058‚Üí  1058‚Üí1. **Tileset name in `addTilesetImage` must match Tiled EXACTLY** ‚Äî first arg is the Tiled tileset name, NOT the Phaser texture key. If they differ, you get `null` and a console warning.
  1059‚Üí  1059‚Üí2. **Layer name in `createLayer` must match Tiled EXACTLY.** Group layer children are prefixed `'GroupName/LayerName'`.
  1060‚Üí  1060‚Üí3. **Each layer can only be created ONCE.** Calling `createLayer` with the same layer ID twice returns `null`.
  1061‚Üí  1061‚Üí4. **`setCollision` MUST be called before physics colliders work.** Without marking tiles as collidable, `physics.add.collider()` passes through all tiles.
  1062‚Üí  1062‚Üí5. **TilemapGPULayer requires `generateLayerDataTexture()` after tile edits** ‚Äî changes won't appear otherwise. Orthographic only; single tileset per layer.
  1063‚Üí  1063‚Üí
  1064‚Üí  1064‚Üí### Right way (Tiled JSON + collision + physics)
  1065‚Üí  1065‚Üí```js
  1066‚Üí  1066‚Üípreload() {
  1067‚Üí  1067‚Üí  this.load.tilemapTiledJSON('map', 'assets/map.json');
  1068‚Üí  1068‚Üí  this.load.image('tiles', 'assets/tileset.png');
  1069‚Üí  1069‚Üí}
  1070‚Üí  1070‚Üícreate() {
  1071‚Üí  1071‚Üí  const map = this.add.tilemap('map');
  1072‚Üí  1072‚Üí  const tileset = map.addTilesetImage('TilesetNameInTiled', 'tiles');  // CRITICAL: Tiled name first
  1073‚Üí  1073‚Üí  const ground = map.createLayer('Ground', tileset);                   // CRITICAL: layer name from Tiled
  1074‚Üí  1074‚Üí  ground.setCollisionByProperty({ collides: true });                    // CRITICAL: before physics
  1075‚Üí  1075‚Üí  this.physics.add.collider(player, ground);
  1076‚Üí  1076‚Üí
  1077‚Üí  1077‚Üí  // Object layer ‚Üí sprites
  1078‚Üí  1078‚Üí  const coins = map.createFromObjects('Items', { name: 'coin', key: 'coin-texture' });
  1079‚Üí  1079‚Üí}
  1080‚Üí  1080‚Üí```
  1081‚Üí  1081‚Üí
  1082‚Üí  1082‚Üí### Pitfalls
  1083‚Üí  1083‚Üí- **DO NOT** use "Collection of Images" tilesets ‚Äî Tiled parser requires single image per tileset. Embedded tilesets required.
  1084‚Üí  1084‚Üí- **DO NOT** expect `setTileIndexCallback` to fire without an active physics collider/overlap.
  1085‚Üí  1085‚Üí- **DO NOT** pass `x, y` to `createLayer` thinking they default to (0,0) ‚Äî they default to the layer offset defined in Tiled.
  1086‚Üí  1086‚Üí- **DO NOT** use `TilemapGPULayer` for isometric/hex/staggered maps ‚Äî orthographic only.
  1087‚Üí  1087‚Üí- **DO NOT** use multiple tilesets on a single `TilemapGPULayer` ‚Äî single tileset only.
  1088‚Üí  1088‚Üí- Tile index `-1` means empty; methods return `null` by default. Pass `nonNull: true` to get a Tile with index -1.
  1089‚Üí  1089‚Üí- `insertNull: true` in tilemap factory stores `null` for empty cells (saves memory, prevents dynamic placement in empty cells).
  1090‚Üí  1090‚Üí- `TilemapGPULayer` max 4096x4096 tiles, up to 2^23 unique tile IDs.
  1091‚Üí  1091‚Üí

  1094‚Üí  1094‚Üí- Tilemap now supports animated tiles on both CPU and GPU layers.
  1095‚Üí  1095‚Üí- `createLayer` 5th arg `gpu: true` enables GPU layer.

  1099‚Üí  1099‚Üí## 27. curves-and-paths
  1100‚Üí  1100‚Üí
  1101‚Üí  1101‚Üí### Critical rules
  1102‚Üí  1102‚Üí1. **`getPoint(t)` uses raw parameter; `getPointAt(u)` uses arc length (evenly spaced).** On a Path, `getPoint` already accounts for arc length across the whole path ‚Äî but on individual curves, use `getPointAt` for even spacing.
  1103‚Üí  1103‚Üí2. **PathFollower uses a Tween internally.** All tween properties (`ease`, `delay`, `repeat`, `yoyo`, callbacks) work. The tween is `persist: true` automatically.
  1104‚Üí  1104‚Üí3. **`positionOnPath: false` (default)** ‚Äî follower's current position becomes the offset from path start. With `true`, the follower snaps to path start and offset is zeroed.
  1105‚Üí  1105‚Üí4. **Ellipse curve angles are in DEGREES** ‚Äî constructor and `startAngle`/`endAngle` accept degrees. Internally stored as radians. `rotation` is radians; `angle` is degrees.
  1106‚Üí  1106‚Üí5. **`cubicBezierTo` parameter order with numbers is (endX, endY, cp1X, cp1Y, cp2X, cp2Y)** ‚Äî end point FIRST, not control points. With Vector2 objects, it's `(cp1, cp2, endPoint)`.
  1107‚Üí  1107‚Üí
  1108‚Üí  1108‚Üí### Right way (path + follower)
  1109‚Üí  1109‚Üí```js
  1110‚Üí  1110‚Üíconst path = this.add.path(100, 200);
  1111‚Üí  1111‚Üípath.lineTo(400, 400);
  1112‚Üí  1112‚Üípath.splineTo([new Phaser.Math.Vector2(600, 300), new Phaser.Math.Vector2(700, 200)]);
  1113‚Üí  1113‚Üí
  1114‚Üí  1114‚Üíconst enemy = this.add.follower(path, 100, 200, 'enemy');
  1115‚Üí  1115‚Üíenemy.startFollow({
  1116‚Üí  1116‚Üí  duration: 3000,
  1117‚Üí  1117‚Üí  positionOnPath: true,    // snap to path start
  1118‚Üí  1118‚Üí  rotateToPath: true,      // auto-rotate to face direction
  1119‚Üí  1119‚Üí  rotationOffset: 90,      // degrees
  1120‚Üí  1120‚Üí  repeat: -1,
  1121‚Üí  1121‚Üí  yoyo: true,
  1122‚Üí  1122‚Üí});
  1123‚Üí  1123‚Üí
  1124‚Üí  1124‚Üí// Draw the path:
  1125‚Üí  1125‚Üíconst gfx = this.add.graphics();
  1126‚Üí  1126‚Üígfx.lineStyle(2, 0xffffff, 1);
  1127‚Üí  1127‚Üípath.draw(gfx, 64);  // 64 points per curve
  1128‚Üí  1128‚Üí```
  1129‚Üí  1129‚Üí
  1130‚Üí  1130‚Üí### Pitfalls
  1131‚Üí  1131‚Üí- **DO NOT** expect `moveTo()` to draw ‚Äî it creates an inactive pseudo-curve that only repositions the end point.
  1132‚Üí  1132‚Üí- **DO NOT** expect cached lengths to update when modifying curve control points ‚Äî call `path.updateArcLengths()`.
  1133‚Üí  1133‚Üí- **DO NOT** confuse `closePath()` (adds an explicit Line curve) with `autoClose = true` (only affects `getPoints` output).
  1134‚Üí  1134‚Üí- **DO NOT** use Spline with fewer than 4 points ‚Äî Catmull-Rom interpolation works best with 4+.
  1135‚Üí  1135‚Üí- Line curve overrides `arcLengthDivisions` to 1 (uniform by nature).
  1136‚Üí  1136‚Üí

  1138‚Üí  1138‚Üí- No major changes.

  1142‚Üí  1142‚Üí## 28. actions-and-utilities
  1143‚Üí  1143‚Üí
  1144‚Üí  1144‚Üí### Critical rules
  1145‚Üí  1145‚Üí1. **Actions operate on PLAIN ARRAYS, not Groups directly.** Always call `group.getChildren()` first.
  1146‚Üí  1146‚Üí2. **`PlaceOn*`/`Random*` actions need `Phaser.Geom` objects**, not Game Object shapes. For a `Phaser.GameObjects.Circle`, pass `circle.geom`.
  1147‚Üí  1147‚Üí3. **`SetXY` defaults `y` to `x`** if `y` is `undefined` or `null`. Pass `0` explicitly if you want y=0 and x=something else.
  1148‚Üí  1148‚Üí4. **`step` is multiplied by iteration index, not added cumulatively.** Item 0 gets `value + 0*step`, item 1 gets `value + 1*step`, etc.
  1149‚Üí  1149‚Üí5. **`Spread` with a single item places it at the midpoint `(min+max)/2`, NOT at `min`.**
  1150‚Üí  1150‚Üí
  1151‚Üí  1151‚Üí### Right way (grid align + scatter + wrap)
  1152‚Üí  1152‚Üí```js
  1153‚Üí  1153‚Üíconst sprites = [];
  1154‚Üí  1154‚Üífor (let i = 0; i < 20; i++) sprites.push(this.add.sprite(0, 0, 'gem'));
  1155‚Üí  1155‚Üí
  1156‚Üí  1156‚ÜíPhaser.Actions.GridAlign(sprites, {
  1157‚Üí  1157‚Üí  width: 5, height: 4, cellWidth: 64, cellHeight: 64, x: 100, y: 100,
  1158‚Üí  1158‚Üí});
  1159‚Üí  1159‚Üí
  1160‚Üí  1160‚Üí// With a Group:
  1161‚Üí  1161‚Üíconst enemies = this.add.group({ key: 'enemy', repeat: 9 });
  1162‚Üí  1162‚ÜíPhaser.Actions.PlaceOnCircle(enemies.getChildren(), new Phaser.Geom.Circle(400, 300, 200));
  1163‚Üí  1163‚Üí
  1164‚Üí  1164‚Üí// Step parameter for staggered values:
  1165‚Üí  1165‚ÜíPhaser.Actions.SetX(sprites, 100, 50);  // x: 100, 150, 200, 250...
  1166‚Üí  1166‚Üí```
  1167‚Üí  1167‚Üí
  1168‚Üí  1168‚Üí### Pitfalls
  1169‚Üí  1169‚Üí- **DO NOT** pass a Group to Actions ‚Äî they expect arrays. Use `group.getChildren()`.
  1170‚Üí  1170‚Üí- **DO NOT** pass a `Phaser.GameObjects.Circle` to `PlaceOnCircle` ‚Äî pass `circle.geom` (a `Phaser.Geom.Circle`).
  1171‚Üí  1171‚Üí- **DO NOT** use `Shuffle` expecting a new array ‚Äî it modifies in place and returns the same array.
  1172‚Üí  1172‚Üí- **DO NOT** use `GetFastValue` with a dot-path key ‚Äî it's top-level only. Use `GetValue` for dot paths.
  1173‚Üí  1173‚Üí- `AddEffectBloom` / `AddEffectShine` / `AddMaskShape` are v4-only filter-based effects ‚Äî they return arrays of created effects, not the input array.
  1174‚Üí  1174‚Üí
  1175‚Üí  1175‚Üí### v3‚Üív4
  1176‚Üí  1176‚Üí- New v4-only effect actions: `AddEffectBloom`, `AddEffectShine`, `AddMaskShape`.
  1177‚Üí  1177‚Üí- New `FitToRegion` action.
